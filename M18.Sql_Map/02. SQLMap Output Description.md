# SQLMap Output Description (CPTS Notes)

## URL Content is Stable

```text
target URL content is stable
```

SQLMap sent multiple identical requests and received similar responses. Stable pages make SQL Injection detection easier because changes caused by payloads are easier to identify.

---

## Parameter Appears Dynamic

```text
GET parameter 'id' appears to be dynamic
```

SQLMap changed the parameter value and noticed a different response.

Example:

```text
?id=1 → Product A
?id=2 → Product B
```

A dynamic parameter is usually processed by the application and is a good SQLi candidate.

---

## Parameter Might Be Injectable

```text
heuristic (basic) test shows that GET parameter 'id' might be injectable
```

SQLMap detected behavior consistent with SQL Injection, usually through database errors.

This is only an indication, not confirmation.

---

## Parameter Might Be Vulnerable to XSS

```text
heuristic (XSS) test shows that GET parameter 'id' might be vulnerable to cross-site scripting (XSS) attacks
```

SQLMap noticed possible reflected input and warns about a potential XSS vulnerability.

---

## Back-End DBMS Identified

```text
it looks like the back-end DBMS is 'MySQL'
```

SQLMap fingerprinted the database and can now focus on DBMS-specific payloads, making testing faster and more accurate.

---

## Level and Risk Values

```text
include all tests for MySQL extending provided level (1) and risk (1)?
```

### Level

Controls how many tests SQLMap performs.

```text
1 = Basic
5 = Extensive
```

### Risk

Controls how aggressive payloads are.

```text
1 = Safe
3 = Aggressive
```

---

## Reflective Values Found

```text
reflective value(s) found and filtering out
```

The application is reflecting SQLMap's payload back in the response.

SQLMap removes this noise before comparing responses.

---

## Parameter Appears Injectable

```text
GET parameter 'id' appears to be 'AND boolean-based blind' injectable
```

SQLMap found strong evidence that the parameter is vulnerable using a specific SQLi technique.

Example:

```sql
AND 1=1
```

vs

```sql
AND 1=2
```

---

## String-Based Detection

```text
with --string="luther"
```

SQLMap uses a constant string in the response to distinguish TRUE and FALSE conditions.

This provides highly reliable detection.

---

## Time-Based Statistical Model

```text
time-based comparison requires a larger statistical model
```

SQLMap collects normal response times and compares them with delayed responses to identify Time-Based Blind SQL Injection.

---

## Extending UNION Tests

```text
automatically extending ranges for UNION query injection technique tests
```

Since another SQLi technique was already found, SQLMap performs more UNION-based testing to increase the chance of finding a faster exploitation method.
