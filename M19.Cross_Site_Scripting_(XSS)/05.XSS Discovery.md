# XSS Discovery

## Overview

Finding XSS vulnerabilities can be as challenging as exploiting them. Discovery methods generally fall into two categories:

* **Automated Discovery** using scanners and tools
* **Manual Discovery** through payload testing and code review

---

# Automated Discovery

Most vulnerability scanners can detect XSS, including:

* Burp Suite Pro
* Nessus
* OWASP ZAP
* XSStrike
* BruteXSS
* XSSer

### How They Work

**Passive Scanning**

* Reviews JavaScript and page source
* Looks for dangerous DOM sinks and sources
* Identifies potential DOM XSS

**Active Scanning**

* Sends multiple XSS payloads
* Checks if payloads are reflected in responses
* Attempts to trigger JavaScript execution

### XSStrike Example

```bash
git clone https://github.com/s0md3v/XSStrike.git

cd XSStrike

pip install -r requirements.txt

python xsstrike.py -u "http://SERVER_IP:PORT/index.php?task=test"
```

XSStrike automatically:

* Finds parameters
* Detects reflections
* Generates custom payloads
* Identifies possible XSS vectors

> Always manually verify scanner findings. Reflection does not always mean executable XSS.

# Manual Discovery

## Payload Testing

The most common approach is testing XSS payloads directly.

Basic payload:

```html
<script>alert(window.origin)</script>
```

Alternative payloads:

```html
<plaintext>
```

```html
<script>print()</script>
```

```html
<img src="" onerror=alert(window.origin)>
```

If JavaScript executes, the application is likely vulnerable.

---

# Why Many Payloads Fail

Different injection points require different payloads.

Examples:

* HTML Context
* Attribute Context
* JavaScript Context
* CSS Context

A payload working in one context may completely fail in another.

---

# Code Review

The most reliable XSS discovery technique.

### Front-End Review

Look for Sources:

```javascript
location.search
location.hash
document.URL
document.cookie
```

Look for Sinks:

```javascript
innerHTML
outerHTML
document.write()
```

If user-controlled input flows from a Source to a Sink without sanitization, XSS may exist.

### Back-End Review

Check:

* Input validation
* Output encoding
* Sanitization
* Template rendering

---

# XSS Discovery Methodology

```text
1. Find Input Fields
2. Find URL Parameters
3. Check Headers & Cookies
4. Test Reflection
5. Identify Context
6. Try Basic Payloads
7. Review Response
8. Review Source Code
9. Identify Sources & Sinks
10. Confirm Execution
```

---

# CPTS Quick Notes

```text
Automated Tools → Find Potential XSS
Manual Testing → Verify XSS
Code Review → Find Hidden XSS

Source
  ↓
User Input
  ↓
Unsafe Sink
  ↓
JavaScript Execution
  ↓
XSS
```

**Remember:** Automated scanners help locate potential vulnerabilities, but manual verification and code review are the most reliable ways to confirm real XSS vulnerabilities.
