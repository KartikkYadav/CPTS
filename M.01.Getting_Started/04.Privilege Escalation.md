# Privilege Escalation (PrivEsc)

## 📌 Overview
Initial access to a system usually provides **low-privileged user access**.  
Privilege Escalation (PrivEsc) is the process of gaining higher privileges:

- **Linux → root**
- **Windows → Administrator / SYSTEM**

This is achieved by exploiting **local vulnerabilities or misconfigurations**.

---

## 🧠 PrivEsc Checklists

After gaining access, perform **thorough enumeration**.

### 📚 Useful Resources
- HackTricks (Linux & Windows checklists)
- PayloadsAllTheThings

### 🎯 Goal
Identify:
- Misconfigurations
- Weak permissions
- Vulnerable services

## ⚙️ Enumeration Scripts

### 🔧 Common Tools

| OS       | Tools |
|----------|------|
| Linux    | LinEnum, linuxprivchecker |
| Windows  | Seatbelt, JAWS |
| Both     | PEASS (LinPEAS / WinPEAS) |

### ⚠️ Important Note
- Scripts generate **a lot of noise**
- May trigger:
  - Antivirus (AV)
  - EDR / Monitoring alerts

👉 In sensitive environments, prefer **manual enumeration**

## 🐧 Example: LinPEAS

- Collects:
  - OS version
  - Users & groups
  - Vulnerabilities

### Output Indicators
- 🔴 Red → High probability PrivEsc vector
- 🟡 Yellow → Potential vulnerability

## ⚡ Sudo Exploitation

### 🔍 Check Permissions
    sudo -l

### 🧪 Example: Full Access
    (ALL : ALL) ALL

👉 Gain root:
    sudo su -

---

### 🚫 NOPASSWD Scenario
    (user : user) NOPASSWD: /bin/echo

👉 Execute without password:
    sudo -u user /bin/echo Hello

### 🔥 Exploitation Resources
- GTFOBins (Linux)
- LOLBAS (Windows)

## ⏰ Scheduled Tasks / Cron Jobs

### 🐧 Linux Locations
- /etc/crontab
- /etc/cron.d
- /var/spool/cron/crontabs/root

### 🎯 Attack Methods
- Add new cron jobs
- Inject malicious scripts

👉 Example:
- Add reverse shell in writable cron directory

## 🔐 Exposed Credentials

### 🔍 Common Locations
- Configuration files
- Log files
- History files:
  - .bash_history (Linux)
  - PSReadLine (Windows)

### 🧪 Example
    config.php → password123

### 🔁 Password Reuse Attack
    su -
    Password: password123

👉 Can lead to **root access**

---

## 🔑 SSH Keys Abuse

### 📖 Read Access

#### Sensitive Files
- /home/user/.ssh/id_rsa
- /root/.ssh/id_rsa

#### Login Using Key
    ssh root@target -i id_rsa

#### Fix Permissions
    chmod 600 id_rsa

---

### ✍️ Write Access Attack

#### Step 1: Generate Key
    ssh-keygen -f key

#### Step 2: Add Public Key
    echo "public_key" >> /root/.ssh/authorized_keys

#### Step 3: Login
    ssh root@target -i key

## 🚀 Key Takeaways

- Enumeration is **critical**
- Focus on:
  - Kernel exploits
  - Misconfigurations
  - Weak permissions
  - Credential leaks
- Use:
  - Automated tools + Manual techniques
- Be cautious:
  - Avoid noisy actions in real environments
