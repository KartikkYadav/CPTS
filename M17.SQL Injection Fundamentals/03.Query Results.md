# Query Results

After learning how to use SQL statements like:
- INSERT
- SELECT
- UPDATE
- ALTER

the next important step is understanding how to control and filter query results. In real-world databases, queries may return thousands of rows, so SQL provides several clauses that help:
- Sort data
- Limit returned records
- Filter specific rows
- Match patterns

These concepts are also extremely important in SQL Injection because attackers frequently manipulate query results using clauses like:
```text
ORDER BY
LIMIT
WHERE
LIKE
```

---

# Sorting Results with ORDER BY

The `ORDER BY` clause is used to sort query results based on one or more columns.

Basic syntax:

```sql id="a4r2pn"
SELECT * FROM table_name ORDER BY column_name;
```

---

# Example — Sorting by Password

```sql id="y1bz2g"
SELECT * FROM logins ORDER BY password;
```

### Example Output

```text id="xg3a5h"
+----+---------------+------------+
| id | username      | password   |
+----+---------------+------------+
| 2  | administrator | adm1n_p@ss |
| 3  | john          | john123!   |
| 1  | admin         | p@ssw0rd   |
| 4  | tom           | tom123!    |
+----+---------------+------------+
```

The results are sorted alphabetically based on the:
```text
password
```
column.

---

# Ascending and Descending Sorting

By default:
```text
ORDER BY
```

sorts in:
```text
Ascending order (ASC)
```

This means:
- Numbers increase from small to large
- Text sorts alphabetically A → Z

---

# DESC Keyword

To reverse sorting order, use:

```sql id="wocvfr"
DESC
```

Example:

```sql id="avfwnj"
SELECT * FROM logins ORDER BY password DESC;
```

### Example Output

```text id="h7u4gg"
+----+---------------+------------+
| id | username      | password   |
+----+---------------+------------+
| 4  | tom           | tom123!    |
| 1  | admin         | p@ssw0rd   |
| 3  | john          | john123!   |
| 2  | administrator | adm1n_p@ss |
+----+---------------+------------+
```

Now the results are sorted:
```text
Z → A
```

---

# Sorting by Multiple Columns

SQL also allows sorting using multiple columns.

Example:

```sql id="jc67xy"
SELECT * FROM logins ORDER BY password DESC, id ASC;
```

### Explanation

| Sort Level | Column | Direction |
|---|---|---|
| Primary Sort | password | DESC |
| Secondary Sort | id | ASC |

---

# Why Multiple Sorting Matters

If multiple rows have identical values in one column:
```text
Secondary sorting helps organize duplicates
```

For example:
- Several users may have the same password
- IDs can then determine their order

---

# LIMIT Clause

Large queries may return huge numbers of rows.

The:
```text
LIMIT
```

clause restricts how many records are returned.

Basic syntax:

```sql id="e0v5vr"
SELECT * FROM table_name LIMIT number;
```

---

# Example — Limiting Results

```sql id="a2gndd"
SELECT * FROM logins LIMIT 2;
```

### Example Output

```text id="tmmsfe"
+----+---------------+------------+
| id | username      | password   |
+----+---------------+------------+
| 1  | admin         | p@ssw0rd   |
| 2  | administrator | adm1n_p@ss |
+----+---------------+------------+
```

Only:
```text
2 rows
```

are returned.

---

# LIMIT with OFFSET

We can also specify:
- Starting position
- Number of rows

Syntax:

```sql id="2pcq4h"
SELECT * FROM table_name LIMIT offset, count;
```

---

# Example — Offset Usage

```sql id="1f7brm"
SELECT * FROM logins LIMIT 1, 2;
```

### Example Output

```text id="kv06ev"
+----+---------------+------------+
| id | username      | password   |
+----+---------------+------------+
| 2  | administrator | adm1n_p@ss |
| 3  | john          | john123!   |
+----+---------------+------------+
```

---

# Understanding OFFSET

The first number:
```text
1
```

means:
```text
Skip the first row
```

The second number:
```text
2
```

means:
```text
Return the next 2 rows
```

---

# Important LIMIT Note

OFFSET counting starts from:
```text
0
```

Example:

| Offset | Starting Row |
|---|---|
| 0 | First row |
| 1 | Second row |
| 2 | Third row |

---

# WHERE Clause

The `WHERE` clause filters records based on conditions.

Basic syntax:

```sql id="xprmd0"
SELECT * FROM table_name WHERE condition;
```

This only returns rows that satisfy the specified condition.

---

# Example — Numeric Filtering

```sql id="0g1eao"
SELECT * FROM logins WHERE id > 1;
```

### Example Output

```text id="qkigjk"
+----+---------------+------------+
| id | username      | password   |
+----+---------------+------------+
| 2  | administrator | adm1n_p@ss |
| 3  | john          | john123!   |
| 4  | tom           | tom123!    |
+----+---------------+------------+
```

Rows where:
```text
id > 1
```

are returned.

The row with:
```text
id = 1
```

is excluded.

---

# Example — String Filtering

```sql id="k0z2o6"
SELECT * FROM logins WHERE username = 'admin';
```

### Output

```text id="b42yur"
+----+----------+----------+
| id | username | password |
+----+----------+----------+
| 1  | admin    | p@ssw0rd |
+----+----------+----------+
```

Only the row where:
```text
username = admin
```

is returned.

---

# Important Data Type Note

Strings and dates should be wrapped inside:
```text
Single quotes (' ')
```

or:
```text
Double quotes (" ")
```

Examples:

```sql id="vh6x9z"
WHERE username = 'admin'
```

```sql id="k9c10p"
WHERE date = '2024-01-01'
```

Numbers usually do not require quotes:

```sql id="6nq0c5"
WHERE id = 1
```

---

# LIKE Clause

The `LIKE` clause searches for patterns instead of exact matches.

This is extremely useful for:
- Searching usernames
- Filtering emails
- Matching partial values

Basic syntax:

```sql id="1j5c6z"
SELECT * FROM table_name WHERE column LIKE 'pattern';
```

---

# Wildcard Characters

SQL uses special wildcard characters with LIKE.

| Wildcard | Meaning |
|---|---|
| `%` | Matches zero or more characters |
| `_` | Matches exactly one character |

---

# Example — Matching Usernames Starting with admin

```sql id="v1qg8j"
SELECT * FROM logins WHERE username LIKE 'admin%';
```

### Example Output

```text id="yy3kbf"
+----+---------------+
| id | username      |
+----+---------------+
| 1  | admin         |
| 4  | administrator |
+----+---------------+
```

---

# Understanding %

The wildcard:
```text
%
```

matches:
```text
Any number of characters
```

So:

```sql id="5eqh8g"
'admin%'
```

matches:
- admin
- administrator
- admin123
- admin_test

---

# Matching Exact Character Length

The underscore:
```text
_
```

matches:
```text
Exactly one character
```

---

# Example — Usernames with 3 Characters

```sql id="5tzjlwm"
SELECT * FROM logins WHERE username LIKE '___';
```

### Example Output

```text id="5f5r1l"
+----+----------+
| id | username |
+----+----------+
| 3  | tom      |
+----+----------+
```

Because:
```text
tom
```

contains:
```text
Exactly 3 characters
```

---

# Why These Clauses Matter in SQL Injection

Clauses like:
- ORDER BY
- LIMIT
- WHERE
- LIKE

are heavily abused during SQL Injection attacks.

Attackers commonly use them to:
- Enumerate columns
- Filter database results
- Extract usernames
- Identify table structures
- Dump sensitive records

Examples:
- `ORDER BY` helps discover column count
- `LIMIT` extracts records one at a time
- `WHERE` filters sensitive rows
- `LIKE` searches partial data

Understanding these SQL query controls is extremely important for:
- CPTS
- SQL Injection
- Database enumeration
- Web exploitation
- Authentication bypass attacks
