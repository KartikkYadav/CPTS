# SQL Statements

After learning how to connect to MySQL, create databases, and create tables, the next important step is understanding the most commonly used SQL statements. These statements are used daily by database administrators, developers, and penetration testers to interact with databases.

The most important SQL statements include:
- INSERT
- SELECT
- DROP
- ALTER
- UPDATE

Understanding these statements is extremely important because SQL Injection attacks often manipulate them directly.

---

# INSERT Statement

The `INSERT` statement is used to add new records into a table.

General syntax:

```sql id="pwzxwv"
INSERT INTO table_name VALUES (column1_value, column2_value, column3_value, ...);
```

This syntax requires values for:
```text id="3hb9cf"
ALL columns in the table
```

---

# Example — Inserting Data

```sql id="v3prgb"
INSERT INTO logins VALUES(1, 'admin', 'p@ssw0rd', '2020-07-02');
```

### Example Output

```text id="p3w0vt"
Query OK, 1 row affected
```

This inserts:
- ID = 1
- Username = admin
- Password = p@ssw0rd
- Date = 2020-07-02

---

# Inserting Specific Columns

Sometimes we do not want to provide values for every column.

For example:
- `AUTO_INCREMENT` fields generate automatically
- `DEFAULT` values populate automatically

In such cases, we specify only selected columns.

Syntax:

```sql id="o9crbw"
INSERT INTO table_name(column2, column3) VALUES(value2, value3);
```

---

# Example — Selective Insert

```sql id="m1m5ja"
INSERT INTO logins(username, password) VALUES('administrator', 'adm1n_p@ss');
```

This skips:
- `id`
- `date_of_joining`

because:
- `id` uses AUTO_INCREMENT
- `date_of_joining` uses DEFAULT NOW()

---

# Important Note — NOT NULL

If a column uses:
```sql id="q0gjmx"
NOT NULL
```

then it cannot be skipped unless a default value exists.

Otherwise MySQL returns an error.

---

# Important Security Note

The examples store passwords in:
```text id="qdl9oa"
Cleartext
```

This is extremely insecure.

Real applications should always:
- Hash passwords
- Encrypt sensitive data

Common password hashing methods:
- bcrypt
- Argon2
- PBKDF2

---

# Inserting Multiple Records

MySQL allows inserting multiple rows in one query.

Example:

```sql id="i65h0d"
INSERT INTO logins(username, password)
VALUES
('john', 'john123!'),
('tom', 'tom123!');
```

### Output

```text id="dt32v9"
Records: 2
Duplicates: 0
Warnings: 0
```

This inserts:
- john
- tom

in a single query.

---

# SELECT Statement

The `SELECT` statement is one of the most important SQL statements.

It is used to:
- Retrieve data
- Read table contents
- Extract records
- Query databases

Most SQL Injection attacks abuse:
```text id="jlwmvi"
SELECT queries
```

---

# Selecting All Data

Syntax:

```sql id="n0pm7m"
SELECT * FROM table_name;
```

The wildcard:
```text id="m2d4gv"
*
```

means:
```text id="8p9sw2"
Select all columns
```

---

# Example — View Entire Table

```sql id="f5knan"
SELECT * FROM logins;
```

### Example Output

```text id="3qh8mh"
+----+---------------+------------+---------------------+
| id | username      | password   | date_of_joining     |
+----+---------------+------------+---------------------+
| 1  | admin         | p@ssw0rd   | 2020-07-02          |
| 2  | administrator | adm1n_p@ss | 2020-07-02          |
| 3  | john          | john123!   | 2020-07-02          |
| 4  | tom           | tom123!    | 2020-07-02          |
+----+---------------+------------+---------------------+
```

This displays every:
- Row
- Column
- Record

inside the table.

---

# Selecting Specific Columns

Instead of retrieving everything, we can select only specific fields.

Syntax:

```sql id="0hsgjd"
SELECT column1, column2 FROM table_name;
```

---

# Example — Selecting Usernames and Passwords

```sql id="hjlwmz"
SELECT username,password FROM logins;
```

### Output

```text id="hzw8gq"
+---------------+------------+
| username      | password   |
+---------------+------------+
| admin         | p@ssw0rd   |
| administrator | adm1n_p@ss |
| john          | john123!   |
| tom           | tom123!    |
+---------------+------------+
```

Only:
- username
- password

columns are displayed.

---

# DROP Statement

The `DROP` statement permanently deletes:
- Tables
- Databases

---

# Dropping a Table

Example:

```sql id="8i7xmk"
DROP TABLE logins;
```

### Output

```text id="d8i63s"
Query OK
```

---

# Verifying Table Removal

```sql id="8lx7zc"
SHOW TABLES;
```

### Output

```text id="6tlv9k"
Empty set
```

The table has been completely removed.

---

# Important Warning — DROP

`DROP` is:
```text id="rqxnpq"
Permanent
```

There is:
- No confirmation
- No recycle bin
- No undo

Once executed:
```text id="8h2d94"
The data is gone
```

unless backups exist.

---

# ALTER Statement

The `ALTER` statement modifies:
- Table structure
- Column names
- Data types
- Existing fields

It is commonly used during:
- Database maintenance
- Application updates
- Schema migrations

---

# Adding a Column

Example:

```sql id="3dxfhr"
ALTER TABLE logins ADD newColumn INT;
```

This creates:
```text id="8zmmzt"
newColumn
```

with datatype:
```text id="x7mn5p"
INT
```

---

# Renaming a Column

Example:

```sql id="o7w7c4"
ALTER TABLE logins RENAME COLUMN newColumn TO newerColumn;
```

This changes:
```text id="y8z3hn"
newColumn → newerColumn
```

---

# Modifying a Datatype

Example:

```sql id="on1trr"
ALTER TABLE logins MODIFY newerColumn DATE;
```

This changes the datatype to:
```text id="0zy0xv"
DATE
```

---

# Dropping a Column

Example:

```sql id="1v1l1n"
ALTER TABLE logins DROP newerColumn;
```

This permanently removes the column.

---

# UPDATE Statement

The `UPDATE` statement modifies:
```text id="f3bdbs"
Existing records
```

inside a table.

Unlike ALTER:
- ALTER changes table structure
- UPDATE changes table data

---

# UPDATE Syntax

```sql id="3yk68u"
UPDATE table_name
SET column1=value1, column2=value2
WHERE condition;
```

---

# Important Concept — WHERE Clause

The:
```text id="pt2h6v"
WHERE
```

clause determines:
```text id="4s3zql"
Which records are updated
```

Without WHERE:
```text id="pr3lpo"
ALL rows get modified
```

which can be catastrophic.

---

# Example — Updating Passwords

```sql id="zyqrpc"
UPDATE logins
SET password = 'change_password'
WHERE id > 1;
```

### Output

```text id="7l4y8m"
Rows matched: 3
Changed: 3
```

---

# Viewing Updated Records

```sql id="0wwmv6"
SELECT * FROM logins;
```

### Output

```text id="9f2zcg"
+----+---------------+-----------------+
| id | username      | password        |
+----+---------------+-----------------+
| 1  | admin         | p@ssw0rd        |
| 2  | administrator | change_password |
| 3  | john          | change_password |
| 4  | tom           | change_password |
+----+---------------+-----------------+
```

Only records where:
```text id="gwnlkc"
id > 1
```

were modified.

---

# Why SQL Statements Matter for CPTS

Understanding SQL statements is critical for:
- SQL Injection
- Database exploitation
- Web application testing
- Authentication bypass
- Data extraction

Attackers frequently manipulate:
- SELECT queries
- INSERT queries
- UPDATE queries

during SQL Injection attacks.

A strong understanding of SQL helps penetration testers:
- Understand backend logic
- Identify vulnerable queries
- Exploit authentication systems
- Extract sensitive data
- Modify records during exploitation

Without understanding SQL syntax and behavior, advanced SQL Injection becomes significantly more difficult.
