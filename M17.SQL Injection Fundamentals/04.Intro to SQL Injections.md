# Intro to SQL Injections

Now that we understand how MySQL databases and SQL queries work, the next important topic is **SQL Injection (SQLi)**.

SQL Injection is one of the most dangerous and common web vulnerabilities. It occurs when user input is inserted directly into SQL queries without proper validation or sanitization, allowing attackers to manipulate the query and execute unintended SQL commands. :contentReference[oaicite:0]{index=0}

---

# Use of SQL in Web Applications

Modern web applications heavily rely on databases to:
- Store user information
- Handle logins
- Save posts and comments
- Store products and orders
- Retrieve search results

A backend application communicates with a database using SQL queries.

For example, a PHP web application may connect to MySQL like this:

```php
$conn = new mysqli("localhost", "root", "password", "users");
$query = "select * from logins";
$result = $conn->query($query);
```

This query retrieves all records from the `logins` table. :contentReference[oaicite:1]{index=1}

The application can then display the returned data:

```php
while($row = $result->fetch_assoc() ){
    echo $row["name"]."<br>";
}
```

This prints each username line by line on the webpage. :contentReference[oaicite:2]{index=2}

# User Input in SQL Queries

Most applications allow users to search or filter data.

Example:

```php
$searchInput =  $_POST['findUser'];
$query = "select * from logins where username like '%$searchInput'";
$result = $conn->query($query);
```

Here:
- User input is stored in `$searchInput`
- That input is inserted directly into the SQL query

# What is Sanitization?

Sanitization means:
- Filtering dangerous characters
- Escaping special symbols
- Preventing user input from being interpreted as code

Without sanitization, attackers may inject:
- SQL commands
- JavaScript
- System commands

into the application. :contentReference[oaicite:4]{index=4}

---

# What is an Injection?

Injection happens when:
```text
User input is treated as executable code instead of normal data
```

An attacker escapes the intended input boundaries and injects new commands.

Common dangerous characters:
- `'`
- `"`
- `;`
- `--`

If the application does not sanitize input correctly, the injected code gets executed. :contentReference[oaicite:5]{index=5}

---

# SQL Injection (SQLi)

SQL Injection occurs when:
```text
User input is inserted directly into an SQL query without proper filtering
```

Example vulnerable code:

```php
$searchInput =  $_POST['findUser'];
$query = "select * from logins where username like '%$searchInput'";
$result = $conn->query($query);
```

:contentReference[oaicite:6]{index=6}

---

# Understanding the Query

The query structure is:

```sql
select * from logins where username like '%$searchInput'
```

If the user enters:

```text
admin
```

The query becomes:

```sql
select * from logins where username like '%admin'
```

This simply searches for usernames matching "admin". :contentReference[oaicite:7]{index=7}

---

# Injecting SQL Code

Because the application does not sanitize input, an attacker can close the query using a single quote `'`.

Example malicious input:

```text
1'; DROP TABLE users;
```

The application inserts it into the query:

```php
'%1'; DROP TABLE users;'
```

:contentReference[oaicite:8]{index=8}

Final SQL query:

```sql
select * from logins where username like '%1'; DROP TABLE users;'
```

This causes:
- The original query to terminate
- A new malicious query to execute

The injected command:

```sql
DROP TABLE users;
```

deletes the `users` table entirely. :contentReference[oaicite:9]{index=9}

---

# Why Single Quote (`'`) is Important

The single quote allows attackers to:
```text
Escape the boundaries of the original SQL query
```

Example:

```sql
'%$searchInput'
```

Injected input:

```text
1'
```

Now the string closes early, allowing SQL code injection afterward.

This concept is fundamental in SQL Injection attacks.

---

# SQL Syntax Errors

The previous payload creates an error:

```text
Error: near line 1: near "'": syntax error
```

:contentReference[oaicite:10]{index=10}

Why?

Because the final query contains:
```text
An extra trailing single quote
```

Example:

```sql
select * from logins where username like '%1'; DROP TABLE users;'
```

Notice the unmatched quote at the end. :contentReference[oaicite:11]{index=11}

---

# Why SQLi Payloads Need to be Valid

For successful SQL Injection:
- The modified query must remain syntactically valid
- Extra quotes or broken syntax must be fixed

Attackers commonly use:
- SQL comments
- Additional quotes
- Logical operators

to repair the query structure.

This becomes extremely important during:
- Authentication bypass
- UNION attacks
- Blind SQL Injection

# Types of SQL Injection

SQL Injections are categorized based on:
```text
How the attacker retrieves the output
```

:contentReference[oaicite:13]{index=13}

---

# 1. In-Band SQL Injection

The attacker directly sees the query output in the application's response.

This is the easiest type of SQL Injection.

Two main types:

## a. Union-Based SQL Injection

Uses:
```sql
UNION SELECT
```

to combine attacker-controlled output with the original query results.

The attacker must identify:
- Number of columns
- Visible columns

before extracting data.

---

## b. Error-Based SQL Injection

The attacker intentionally triggers database errors to leak information.

Example leaked information:
- Database version
- Table names
- Query output

This works when:
```text
SQL or PHP errors are displayed on the frontend
```

:contentReference[oaicite:14]{index=14}

---

# 2. Blind SQL Injection

Sometimes the application does NOT display query results.

In these situations, attackers retrieve data indirectly.

Two main types:

---

## a. Boolean-Based Blind SQLi

Uses:
- TRUE/FALSE conditions
- Conditional responses

Example:

```sql
AND 1=1
```

vs

```sql
AND 1=2
```

The attacker observes:
- Page changes
- Response differences
- Missing content

to infer information character by character.

---

## b. Time-Based Blind SQLi

Uses time delays such as:

```sql
SLEEP()
```

If the condition is true, the server delays the response.

Example:

```sql
AND IF(1=1,SLEEP(5),0)
```

If the page pauses for 5 seconds:
```text
Condition evaluated to TRUE
```

:contentReference[oaicite:15]{index=15}

---

# 3. Out-of-Band SQL Injection

In some cases:
- No output is displayed
- No errors appear
- Time-based techniques fail

Attackers then force the database to send data externally through:
- DNS requests
- HTTP requests
- SMB connections

This is called:
```text
Out-of-Band SQL Injection
```
