# Custom Wordlists

Pre-made wordlists such as:
- rockyou.txt
- SecLists

are useful during brute-force attacks because they contain millions of common usernames and passwords. However, these lists are:
```text
Generic
```

and may not work efficiently against specific targets or organizations.

For example, if a pentester is targeting:
```text
Thomas Edison
```

using a huge public username list may waste time because the organization could follow a custom username format such as:
- tedison
- thomas.e
- edisont
- t.edison

instead of simple names like:
```text
thomas
```

or:
```text
tedison123
```

This is where:
```text
Custom wordlists
```

become extremely powerful.

Custom wordlists are specially crafted username and password lists created using:
- OSINT
- Social media
- Company information
- Personal details
- Leaked data
- Naming conventions

This focused approach dramatically increases the chances of successful brute-force attacks while reducing unnecessary login attempts.

---

# Username Anarchy

Creating usernames manually can become difficult because users often create:
- Nickname variations
- Initial-based usernames
- Birth year combinations
- Leetspeak modifications
- Hobby-based usernames

For example, a user named:
```text
Jane Smith
```

may use:
- janesmith
- smithj
- jsmith
- jane1990
- j4n3
- smith_golf
- jane_the_hacker

and many other combinations.

To automate this process, we can use:
```text
Username Anarchy
```

which generates realistic username combinations automatically.

---

# Installing Username Anarchy

```bash
sudo apt install ruby -y
git clone https://github.com/urbanadventurer/username-anarchy.git
cd username-anarchy
```

---

# Generating Usernames

```bash
./username-anarchy Jane Smith > jane_smith_usernames.txt
```

This creates a custom username list containing:
- First/last combinations
- Initials
- Dot formats
- Mixed formats
- Shortened usernames
- Capitalized variations

Examples:
```text
janesmith
j.smith
smithj
jsmith
JaneSmith
```

This gives attackers a much more targeted username list compared to generic datasets.

# Password Policy Filtering

Sometimes organizations enforce password policies such as:
- Minimum length
- Uppercase letters
- Numbers
- Special characters

Suppose the target company requires:
- Minimum 6 characters
- One uppercase letter
- One lowercase letter
- One number
- Two special characters

We can filter the generated passwords using grep.

---

# Filtering Passwords

```bash
grep -E '^.{6,}$' jane.txt | \
grep -E '[A-Z]' | \
grep -E '[a-z]' | \
grep -E '[0-9]' | \
grep -E '([!@#$%^&*].*){2,}' \
> jane-filtered.txt
```

This removes passwords that do not meet the company’s password requirements.

The filtering process reduces:
```text
Large noisy wordlists
```

into:
```text
Smaller high-probability password sets
```

which improves brute-force efficiency.

# Using Hydra with Custom Wordlists

After generating:
- Custom usernames
- Filtered passwords

we can launch a targeted Hydra attack.

---

# Hydra Command

```bash
hydra -L jane_smith_usernames.txt \
-P jane-filtered.txt \
IP -s PORT -f \
http-post-form \
"/:username=^USER^&password=^PASS^:Invalid credentials"
```

---

# Command Explanation

| Option | Purpose |
|---|---|
| `-L` | Username wordlist |
| `-P` | Password wordlist |
| `IP` | Target IP |
| `-s PORT` | Target port |
| `-f` | Stop after first success |
| `http-post-form` | Attack web login form |

Hydra replaces:
```text
^USER^
^PASS^
```

with values from the custom wordlists and attempts authentication automatically.

Custom wordlists are significantly more effective because they:
- Match real human behavior
- Reflect naming conventions
- Include personal details
- Reduce unnecessary guesses
- Improve brute-force speed
- Increase attack success rates

In real-world pentesting:
```text
OSINT-driven custom wordlists often outperform massive generic dictionaries
```

because users commonly create passwords based on:
- Personal memories
- Pets
- Dates
- Hobbies
- Relationships
- Favorite things

rather than completely random strings.
