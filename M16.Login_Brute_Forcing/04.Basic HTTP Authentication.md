# Basic HTTP Authentication

Basic HTTP Authentication, commonly called:
```text
Basic Auth
```

is one of the simplest authentication mechanisms used by web applications to protect:
- Sensitive pages
- Admin panels
- APIs
- Restricted resources

Although Basic Auth is very easy to implement, it also has significant security weaknesses, making it a frequent target for:
- Brute-force attacks
- Password spraying
- Credential stuffing

---

# How Basic Authentication Works

Basic Auth follows a:
```text
Challenge-response authentication model
```

The authentication process works as follows:

1. A user attempts to access a protected resource.
2. The web server responds with:
   ```text
   HTTP 401 Unauthorized
   ```
3. The server includes a:
   ```text
   WWW-Authenticate
   ```
   header requesting credentials.
4. The browser displays a login prompt asking for:
   - Username
   - Password
5. The user submits credentials.
6. The browser combines:
   ```text
   username:password
   ```
7. The combined string is encoded using:
   ```text
   Base64
   ```
8. The encoded value is sent inside the:
   ```text
   Authorization
   ```
   header.

---

# Example Basic Auth Request

```http
GET /protected_resource HTTP/1.1
Host: www.example.com
Authorization: Basic YWxpY2U6c2VjcmV0MTIz
```

---

# Understanding the Authorization Header

The following value:

```text
YWxpY2U6c2VjcmV0MTIz
```

is Base64 encoded data.

After decoding:

```text
alice:secret123
```

The server then:
- Decodes the credentials
- Verifies them against its database
- Grants or denies access

---

# Security Weakness of Basic Auth

Basic Auth does:
- NOT encrypt credentials
- Only Base64 encodes them

Base64 is:
```text
Encoding, not encryption
```

Anyone intercepting the traffic can easily decode the credentials if:
- HTTP is used instead of HTTPS
- TLS is misconfigured

Because of this, Basic Auth is considered weak when:
- Used without HTTPS
- Combined with weak passwords
- Exposed to the internet

---

# Exploiting Basic Auth with Hydra

Hydra can brute-force Basic HTTP Authentication using:
```text
http-get
```

or:
```text
http-head
```

modules.

In this scenario:
- The username is already known:
  ```text
  basic-auth-user
  ```
- We only need to brute-force the password.

---

This downloads a commonly used password list from:
```text
SecLists
```

The file contains:
```text
200 commonly used passwords
```

which are often effective against weak authentication systems.

---

# Hydra Command

```bash
hydra -l basic-auth-user -P 2023-200_most_used_passwords.txt 127.0.0.1 http-get / -s 81
```

---

# Breaking Down the Command

## Username

```bash
-l basic-auth-user
```

Specifies the username:
```text
basic-auth-user
```

Since the username is already known, Hydra only needs to brute-force the password.

---

## Password Wordlist

```bash
-P 2023-200_most_used_passwords.txt
```

Tells Hydra to:
- Read passwords from the wordlist
- Test each password one by one

against the target account.

---

## Target IP Address

```bash
127.0.0.1
```

Specifies the target system.

In this case:
```text
localhost
```

---

## HTTP GET Module

```bash
http-get /
```

This tells Hydra:
- The target uses HTTP authentication
- Authentication should be tested against:
  ```text
  /
  ```
  (the root path)

Hydra will send authenticated HTTP GET requests during the attack.

---

## Custom Port

```bash
-s 81
```

Overrides the default HTTP port:
```text
80
```

and tells Hydra to target:
```text
Port 81
```

# Understanding the Output

Hydra:
- Iterates through every password in the wordlist
- Sends authentication requests
- Checks server responses

When Hydra receives:
```text

```

it displays:
- Valid username
- Valid password

and stops the attack if configured accordingly.

---

# Why This Attack Works

This attack succeeds because:
- Weak passwords are used
- No account lockout exists
- No rate limiting exists
- Basic Auth is directly exposed
- Common passwords are accepted

Attackers frequently use:
```text
Password wordlists
```

containing:
- Default passwords
- Common passwords
- Leaked passwords

to exploit weak authentication systems.

---

# Real-World Scenarios

Basic HTTP Authentication is commonly found in:
- Internal admin panels
- Routers
- IoT devices
- APIs
- Development dashboards
- Backup systems

Many systems still:
- Use default credentials
- Use weak passwords
- Expose Basic Auth externally

making them vulnerable to brute-force attacks.

---

# Important Security Recommendations

To secure Basic Authentication:
- Always use HTTPS
- Implement MFA
- Enforce strong passwords
- Use account lockout policies
- Add rate limiting
- Monitor authentication logs
- Disable default credentials

---

# Key Learning Points

- Basic HTTP Authentication uses Base64 encoded credentials inside the Authorization header
- Base64 encoding is not encryption
- Hydra can brute-force Basic Auth using the `http-get` module
- Weak passwords make Basic Auth highly vulnerable to brute-force attacks
- HTTPS is essential when using Basic Authentication
- Hydra systematically tests passwords from a wordlist until valid credentials are discovered
