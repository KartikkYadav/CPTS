# Hydra

Source File: :contentReference[oaicite:0]{index=0}

Hydra is a fast network login cracker that supports numerous attack protocols. It is widely used during penetration testing to brute-force authentication services such as:
- SSH
- FTP
- HTTP login forms
- Databases
- RDP
- VNC
- Email services

Hydra is extremely popular because it is:
- Fast
- Flexible
- Easy to use
- Supports many protocols

It can perform:
- Password spraying
- Credential stuffing
- Brute-force attacks
- Login enumeration

against multiple services.

---

# Why Hydra is Popular

## Speed and Efficiency

Hydra uses:
```text
Parallel connections
```

to perform multiple login attempts simultaneously.

This significantly speeds up brute-force attacks compared to manually testing credentials one at a time.

---

## Flexibility

Hydra supports many protocols and services, making it adaptable for various attack scenarios.

It can target:
- Remote login services
- Web applications
- Databases
- Mail servers
- Remote desktop services

using different modules.

---

## Ease of Use

Hydra has:
- Simple syntax
- Straightforward commands
- Easy configuration

which makes it beginner-friendly while still being powerful enough for advanced attacks.

# Hydra Basic Syntax

```bash
hydra [login_options] [password_options] [attack_options] [service_options]
```

Hydra commands are generally built using:
- Username options
- Password options
- Service options
- Target information

---

# Common Hydra Parameters

| Parameter | Explanation | Example |
|-----------|-------------|---------|
| `-l LOGIN` | Single username | `-l admin` |
| `-L FILE` | Username wordlist | `-L users.txt` |
| `-p PASS` | Single password | `-p password123` |
| `-P FILE` | Password wordlist | `-P passwords.txt` |
| `-t TASKS` | Number of parallel tasks/threads | `-t 4` |
| `-f` | Stop after first valid credential | `-f` |
| `-s PORT` | Specify custom port | `-s 2222` |
| `-v` | Verbose output | `-v` |
| `-V` | Very verbose output | `-V` |
| `service://server` | Specify target service | `ssh://192.168.1.10` |

# FTP Brute Force

Hydra can brute-force FTP services using username and password lists.

## Example

```bash
hydra -l admin -P passwords.txt ftp://192.168.1.100
```

This command:
- Uses username `admin`
- Uses passwords from `passwords.txt`
- Targets FTP service on the target IP

---

# SSH Brute Force

SSH brute forcing is one of Hydra’s most common use cases.

## Example

```bash
hydra -l root -P passwords.txt ssh://192.168.1.100
```

This command:
- Uses username `root`
- Tries passwords from the wordlist
- Targets SSH service

Hydra will attempt every password until:
- A valid credential is found
- Wordlist is exhausted

---

# HTTP Authentication Brute Force

Suppose a website uses:
```text
Basic HTTP Authentication
```

and we have:
- `usernames.txt`
- `passwords.txt`

We can brute-force it using:

```bash
hydra -L usernames.txt -P passwords.txt www.example.com http-get
```

This command instructs Hydra to:
- Use usernames from the username file
- Use passwords from the password file
- Attack the HTTP authentication service

Hydra systematically tests every username-password combination.

---

# Multiple SSH Targets

Hydra can attack multiple systems simultaneously using:
```text
-M
```

which accepts a target list.

---

# Example

```bash
hydra -l root -p toor -M targets.txt ssh
```

This command:
- Uses username `root`
- Uses password `toor`
- Targets every IP listed in `targets.txt`
- Performs SSH brute-force attacks

This is useful during:
- Internal assessments
- Password spraying
- Weak credential testing

---

# FTP on Non-Standard Port

Hydra supports custom ports using:
```text
-s
```

---

# Example

```bash
hydra -L usernames.txt -P passwords.txt -s 2121 -V ftp.example.com ftp
```

This command:
- Targets FTP service
- Uses custom port `2121`
- Enables verbose output using `-V`

Verbose mode helps monitor:
- Current attempts
- Successful logins
- Progress of the attack

---

# Brute-Forcing Web Login Forms

Hydra can also brute-force:
```text
HTML login forms
```

using:
```text
http-post-form
```

This is extremely useful during:
- Web pentesting
- Bug bounty testing
- Login panel assessments

---

# Example Web Login Form Attack

```bash
hydra -l admin -P passwords.txt www.example.com http-post-form "/login:user=^USER^&pass=^PASS^:S=302"
```

---

# Understanding the Command

This command:
- Uses username `admin`
- Uses passwords from the wordlist
- Targets `/login`
- Sends POST requests

Hydra replaces:
```text
^USER^
^PASS^
```

with:
- Username
- Password

during every attempt.

---

# Success Condition

```text
:S=302
```

tells Hydra:
```text
HTTP 302 response indicates successful login
```

Hydra uses:
- Response codes
- Error messages
- Redirects

to determine:
- Success
- Failure

during brute-force attacks.

---

# Advanced RDP Brute Force

Hydra can generate passwords automatically using:
```text
-x
```

instead of requiring a wordlist.

---

# Example

```bash
hydra -l administrator -x 6:8:abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789 192.168.1.100 rdp
```

---

# Understanding the Command

This command:
- Targets RDP service
- Uses username `administrator`
- Generates passwords automatically

Password generation rules:

```text
6:8
```

means:
```text
Minimum 6 characters
Maximum 8 characters
```

Character set includes:
- Lowercase letters
- Uppercase letters
- Numbers

Hydra then generates:
```text
All possible combinations
```

within those constraints.
