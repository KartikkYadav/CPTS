# Password Spraying, Credential Stuffing, and Default Credentials

## Password Spraying
- One password tested against many users
- Helps avoid account lockouts
- Common in Active Directory attacks

### NetExec Example

```bash
netexec smb 10.100.38.0/24 -u users.list -p 'ChangeMe123!'
```

---

# Credential Stuffing

- Uses leaked `username:password` pairs
- Exploits password reuse across services

### Hydra Example

```bash
hydra -C user_pass.list ssh://10.100.38.23
```

---

# Default Credentials

- Many devices/services use default logins
- Common targets:
  - Routers
  - Firewalls
  - Databases
  - Web panels

# Key Points

- Password spraying = 1 password → many users
- Credential stuffing = leaked credentials reuse
- Default passwords are still common
- NetExec useful for SMB spraying
- Hydra useful for stuffing attacks

# One-Line Revision

These attacks exploit weak passwords, credential reuse, and unchanged default credentials to gain access.
