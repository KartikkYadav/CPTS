# 📌 Cracking Protected Files - CPTS Notes

---

# 🔍 Why Protected Files Matter

Sensitive files are often protected using:
- Passwords
- Encryption
- Passphrases

Common protected files:
- PDFs
- Office documents
- ZIP/RAR archives
- SSH keys

Even though companies improve security awareness, employees still:
- Share sensitive files
- Reuse weak passwords
- Use guessable passphrases :contentReference[oaicite:0]{index=0}  

---

# 🔐 Encryption Basics ⭐ IMPORTANT

## 1️⃣ Symmetric Encryption
- Same key used for:
  - Encryption
  - Decryption

Example:
- AES-256

Used for:
- File/folder encryption

---

## 2️⃣ Asymmetric Encryption
- Uses:
  - Public key
  - Private key

Process:
- Encrypt with public key
- Decrypt with private key

Used for:
- Secure file transfer
- SSH
- GPG

---

# 🔍 Hunting for Encrypted Files ⭐ IMPORTANT

Attackers often search systems for:
- Sensitive documents
- Encrypted files
- Password-protected archives

---

# ⭐ MAIN COMMAND

```bash
for ext in $(echo ".xls .xls* .xltx .od* .doc .doc* .pdf .pot .pot* .pp*");do echo -e "\nFile extension: " $ext; find / -name *$ext 2>/dev/null | grep -v "lib\|fonts\|share\|core" ;done
```

---

# 📌 What This Does

Searches for:
- Excel files
- Word documents
- PDFs
- OpenDocument files
- PowerPoint files

Useful during:
- Post-exploitation
- Data discovery

---

# 🔍 Hunting for SSH Keys ⭐ VERY IMPORTANT

SSH private keys usually do NOT have standard extensions.

Instead, search by file content.

---

# ⭐ MAIN COMMAND

```bash
grep -rnE '^\-{5}BEGIN [A-Z0-9]+ PRIVATE KEY\-{5}$' /* 2>/dev/null
```

---

# 📌 Example Output

```text
/home/jsmith/.ssh/id_ed25519
/home/jsmith/.ssh/SSH.private
/home/jsmith/Documents/id_rsa
```

---

# 🔍 Encrypted SSH Keys ⭐ IMPORTANT

Older PEM keys show encryption info.

---

## Example

```text
Proc-Type: 4,ENCRYPTED
DEK-Info: AES-128-CBC
```

---

# 🔍 Check if SSH Key Requires Passphrase ⭐ IMPORTANT

## ⭐ MAIN COMMAND

```bash
ssh-keygen -yf ~/.ssh/id_rsa
```

---

# 📌 Result

### Unencrypted Key
Displays public key immediately.

### Encrypted Key
Prompts for passphrase:
```text
Enter passphrase for "/home/jsmith/.ssh/id_rsa":
```

# 🔧 2john Tools ⭐ VERY IMPORTANT

JtR includes tools for extracting hashes from protected files.

---

# ⭐ MAIN COMMAND

```bash
locate *2john*
```

---

# 📌 Common Tools

| Tool | Purpose |
|------|----------|
| ssh2john | SSH keys |
| zip2john | ZIP archives |
| rar2john | RAR archives |
| office2john | Office docs |
| pdf2john | PDF files |
| keepass2john | KeePass DB |
| bitlocker2john | BitLocker |

---

# 🔓 Cracking Encrypted SSH Keys ⭐ IMPORTANT

## Step 1: Extract Hash

### ⭐ MAIN COMMAND

```bash
ssh2john.py SSH.private > ssh.hash
```

---

## Step 2: Crack Password

### ⭐ MAIN COMMAND

```bash
john --wordlist=rockyou.txt ssh.hash
```

---

## 📌 Example Result

```text
1234 (SSH.private)
```

---

## Step 3: Show Cracked Password

### ⭐ MAIN COMMAND

```bash
john ssh.hash --show
```

# ⚡ CPTS Key Points ⭐

- Search systems for protected files
- SSH keys may contain valuable access
- `2john` tools extract hashes from files
- Cracking process:
  - Extract hash
  - Crack with JtR/Hashcat
- Weak passwords still common

---

# ⚠️ Important Notes

- Strong passwords may be uncrackable
- Longer passwords increase cracking time
- Public wordlists are often insufficient
- Custom wordlists improve success rate

---

# 🧠 One-Line Revision

Protected file cracking involves extracting hashes from encrypted files using `2john` tools and cracking them with JtR or Hashcat.
