# Cracking Protected Files

## Why Protected Files Matter

Sensitive files are often protected using:
- Passwords
- Encryption
- Passphrases

Common protected files include:
- PDFs
- Office documents
- ZIP/RAR archives
- SSH keys

Even with improved security awareness, employees still:
- Share sensitive files
- Reuse weak passwords
- Use guessable passphrases :contentReference[oaicite:0]{index=0}

# Encryption Basics

## Symmetric Encryption
- Same key used for:
  - Encryption
  - Decryption

Example:
- AES-256

Commonly used for:
- File encryption
- Folder encryption

---

## Asymmetric Encryption
- Uses:
  - Public key
  - Private key

Process:
- Encrypt with public key
- Decrypt with private key

Commonly used for:
- Secure file transfer
- SSH
- GPG

---

# Hunting for Encrypted Files

Attackers often search systems for:
- Sensitive documents
- Encrypted files
- Password-protected archives

---

## Main Command

```bash
for ext in $(echo ".xls .xls* .xltx .od* .doc .doc* .pdf .pot .pot* .pp*");do echo -e "\nFile extension: " $ext; find / -name *$ext 2>/dev/null | grep -v "lib\|fonts\|share\|core" ;done
```

---

## Purpose
Searches the system for:
- Excel files
- Word documents
- PDFs
- OpenDocument files
- PowerPoint files

Useful during:
- Post-exploitation
- Sensitive data discovery

# Hunting for SSH Keys

SSH private keys usually do not have standard extensions.

Instead, identify them using file content.

---

## Main Command

```bash
grep -rnE '^\-{5}BEGIN [A-Z0-9]+ PRIVATE KEY\-{5}$' /* 2>/dev/null
```

---

## Example Output

```text
/home/jsmith/.ssh/id_ed25519
/home/jsmith/.ssh/SSH.private
/home/jsmith/Documents/id_rsa
```

---

# Encrypted SSH Keys

Older PEM keys show encryption details.

Example:

```text
Proc-Type: 4,ENCRYPTED
DEK-Info: AES-128-CBC
```

---

# Check if SSH Key Requires Passphrase

## Main Command

```bash
ssh-keygen -yf ~/.ssh/id_rsa
```

---

## Results

### Unencrypted Key
Displays public key immediately.

### Encrypted Key
Prompts for passphrase:

```text
Enter passphrase for "/home/jsmith/.ssh/id_rsa":
```

# 2john Tools

John The Ripper includes tools for extracting hashes from protected files.

---

## Main Command

```bash
locate *2john*
```

---

## Common Tools

| Tool | Purpose |
|------|----------|
| ssh2john | SSH keys |
| zip2john | ZIP archives |
| rar2john | RAR archives |
| office2john | Office documents |
| pdf2john | PDF files |
| keepass2john | KeePass databases |
| bitlocker2john | BitLocker |

---

# Cracking Encrypted SSH Keys

## Step 1: Extract Hash

```bash
ssh2john.py SSH.private > ssh.hash
```

---

## Step 2: Crack Password

```bash
john --wordlist=rockyou.txt ssh.hash
```

---

## Example Result

```text
1234 (SSH.private)
```

---

## Step 3: Show Cracked Password

```bash
john ssh.hash --show
```

---

# Key Points

- Search systems for protected files
- SSH keys may provide direct access
- `2john` tools extract hashes from encrypted files
- Workflow:
  1. Extract hash
  2. Crack with JtR or Hashcat
- Weak passwords remain common

---

# Important Notes

- Strong passwords may not be crackable
- Longer passwords significantly increase cracking time
- Public wordlists are often insufficient
- Custom wordlists improve cracking success
