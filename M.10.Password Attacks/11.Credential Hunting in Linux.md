# Credential Hunting in Linux

## Overview

After gaining access to a Linux system, one of the first goals is:
- Finding credentials
- Escalating privileges

Common credential sources:
- Configuration files
- Databases
- Notes
- Scripts
- Cronjobs
- SSH keys
- History files
- Logs
- Browser credentials
- Memory/cache

---

# Configuration Files

Common extensions:
- `.conf`
- `.config`
- `.cnf`

These files often contain:
- Usernames
- Passwords
- Database credentials

---

## Find Configuration Files

```bash
for l in $(echo ".conf .config .cnf");do find / -name *$l 2>/dev/null | grep -v "lib\|fonts\|share\|core";done
```

---

## Search for Credentials

```bash
for i in $(find / -name *.cnf 2>/dev/null | grep -v "doc\|lib");do grep "user\|password\|pass" $i 2>/dev/null | grep -v "\#";done
```

---

# Databases

Databases may contain:
- Credentials
- Tokens
- User data

## Find Database Files

```bash
for l in $(echo ".sql .db .*db .db*");do find / -name *$l 2>/dev/null;done
```

---

# Notes and Scripts

Users often store passwords inside:
- `.txt` files
- Scripts
- Notes

Scripts may contain:
- Hardcoded credentials
- API keys

---

## Find Notes

```bash
find /home/* -type f -name "*.txt" -o ! -name "*.*"
```

---

## Find Scripts

```bash
for l in $(echo ".py .sh .pl .go");do find / -name *$l 2>/dev/null;done
```

---

# Cronjobs

Cronjobs may run scripts using stored credentials.

Important locations:
- `/etc/crontab`
- `/etc/cron.d/`

---

## View Cronjobs

```bash
cat /etc/crontab
```

---

# History Files

Useful files:
- `.bash_history`
- `.bashrc`

May contain:
- Passwords
- Sensitive commands

---

## Main Command

```bash
tail -n5 /home/*/.bash*
```

---

# Log Files

Useful logs:

| File | Purpose |
|------|----------|
| `/var/log/auth.log` | Authentication logs |
| `/var/log/syslog` | System logs |
| `/var/log/cron` | Cron activity |

---

## Search Interesting Logs

```bash
grep "accepted\|failed\|sudo\|ssh" /var/log/*
```

---

# Memory and Cache

Applications may store credentials in:
- Memory
- Cache
- Sessions

---

# Mimipenguin

Extracts credentials from memory.

## Main Command

```bash
sudo python3 mimipenguin.py
```

---

# LaZagne

Extracts:
- Browser passwords
- WiFi credentials
- SSH keys
- Stored passwords

## Main Command

```bash
sudo python2.7 laZagne.py all
```

---

# Browser Credentials

Firefox stores credentials in:

```text
.mozilla/firefox/
```

Important file:
```text
logins.json
```

---

## View Stored Credentials

```bash
cat .mozilla/firefox/*.default-release/logins.json | jq .
```

---

# Firefox Decrypt

Decrypts saved Firefox passwords.

## Main Command

```bash
python3.9 firefox_decrypt.py
```

---

# Key Points

- Config files commonly leak credentials
- Scripts may contain hardcoded passwords
- `.bash_history` often exposes secrets
- Browser credential stores are valuable
- LaZagne and Mimipenguin automate credential extraction

---

# One-Line Revision

Credential hunting in Linux involves searching files, logs, scripts, browser storage, and memory for passwords and authentication secrets.
