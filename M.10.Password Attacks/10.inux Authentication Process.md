# Linux Authentication Process

Linux systems commonly use:

```text
PAM (Pluggable Authentication Modules)
```

PAM is responsible for:
- User authentication
- Password management
- Session handling

Common PAM modules:
- `pam_unix.so`
- `pam_unix2.so`

Main authentication files:
- `/etc/passwd`
- `/etc/shadow`

:contentReference[oaicite:0]{index=0}

---

# /etc/passwd File

The `/etc/passwd` file stores information about every user account on the system.

It is:
- World-readable
- Used by applications and services

---

# Example Entry

```text
htb-student:x:1000:1000:,,,:/home/htb-student:/bin/bash
```

---

# Important Fields

| Field | Description |
|------|-------------|
| Username | User account name |
| Password | Usually contains `x` |
| UID | User ID |
| GID | Group ID |
| Home Directory | User's home folder |
| Shell | Default login shell |

---

# Password Field

Usually contains:

```text
x
```

Meaning:
- Password hashes are stored in `/etc/shadow`

Older or misconfigured systems may store password hashes directly inside `/etc/passwd`.

---

# Dangerous Misconfiguration

If `/etc/passwd` becomes writable, an attacker may remove the root password field.

Example:

```text
root::0:0:root:/root:/bin/bash
```

This allows:

```bash
su
```

without entering a password.

---

# /etc/shadow File

The `/etc/shadow` file stores:
- Password hashes
- Password policy information
- Password aging details

Unlike `/etc/passwd`, it is only readable by privileged users.

---

# Example Entry

```text
htb-student:$y$j9T$HASH:18955:0:99999:7:::
```

---

# Password Hash Format

```text
$<id>$<salt>$<hash>
```

---

# Common Hash Algorithms

| ID | Algorithm |
|----|------------|
| 1 | MD5 |
| 5 | SHA-256 |
| 6 | SHA-512 |
| y | Yescrypt |

Modern Linux systems commonly use:
```text
yescrypt
```

Older systems may still use weaker hashes like:
- MD5
- SHA1

---

# Special Password Values

| Value | Meaning |
|------|----------|
| `!` | Password login disabled |
| `*` | Account locked |
| Empty | No password required |

---

# /etc/security/opasswd

This file stores:
- Old password hashes

Used by PAM to:
- Prevent password reuse

---

# Example

```bash
sudo cat /etc/security/opasswd
```

---

# Important Observation

Example hash:

```text
$1$
```

Means:
```text
MD5
```

MD5 is:
- Weak
- Fast to crack

Old passwords are useful because users often reuse:
- Similar passwords
- Similar patterns

Example:
```text
Summer2023!
Summer2024!
```

# Cracking Linux Password Hashes

Once root access is obtained:
1. Copy passwd and shadow files
2. Combine them using `unshadow`
3. Crack hashes offline

---

# Backup Files

## Main Commands

```bash
sudo cp /etc/passwd /tmp/passwd.bak
sudo cp /etc/shadow /tmp/shadow.bak
```

---

# Combine Files with unshadow

## Main Command

```bash
unshadow /tmp/passwd.bak /tmp/shadow.bak > /tmp/unshadowed.hashes
```

Purpose:
- Creates a crackable hash file for:
  - John The Ripper
  - Hashcat

---

# Crack Hashes with Hashcat

## Main Command

```bash
hashcat -m 1800 -a 0 /tmp/unshadowed.hashes rockyou.txt
```

---

# Important Hashcat Mode

```text
1800 = SHA-512 crypt
```

---

# Command Breakdown

| Option | Meaning |
|------|----------|
| `-m 1800` | SHA-512 crypt |
| `-a 0` | Dictionary attack |
| `rockyou.txt` | Password wordlist |

---

# Key Points

- `/etc/passwd` stores user information
- `/etc/shadow` stores password hashes
- PAM handles Linux authentication
- `unshadow` combines passwd + shadow files
- Root access is required to read shadow file
- Weak hashes crack much faster

---

# One-Line Revision

Linux authentication relies on PAM with `/etc/passwd` and `/etc/shadow`, and attackers with root access can extract and crack password hashes offline using `unshadow` and Hashcat.
