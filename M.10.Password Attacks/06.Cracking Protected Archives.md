# Cracking Protected Archives

## Overview

Besides standalone files, penetration testers often encounter:
- ZIP archives
- RAR files
- GZIP files
- BitLocker encrypted drives
- Virtual disks

Organizations commonly use archives to:
- Organize project files
- Compress multiple documents
- Share data efficiently

Archives may contain:
- Password-protected documents
- Credentials
- Sensitive company information :contentReference[oaicite:0]{index=0}

# Cracking ZIP Files

ZIP archives are heavily used in Windows environments.

The process:
1. Extract hash
2. Crack with John The Ripper

---

## Step 1: Extract ZIP Hash

### Main Command

```bash
zip2john ZIP.zip > zip.hash
```

---

## View Extracted Hash

```bash
cat zip.hash
```

---

## Example Hash

```text
ZIP.zip/customers.csv:$pkzip2$1*2*2*0*...
```

---

## Step 2: Crack ZIP Password

### Main Command

```bash
john --wordlist=rockyou.txt zip.hash
```

---

## Example Result

```text
1234 (ZIP.zip/customers.csv)
```

---

## Step 3: Show Cracked Password

```bash
john zip.hash --show
```

# Cracking OpenSSL Encrypted GZIP Files

Some GZIP files are encrypted using OpenSSL.

It may not be obvious immediately.

---

# Identify File Type

## Main Command

```bash
file GZIP.gzip
```

---

## Example Output

```text
GZIP.gzip: openssl enc'd data with salted password
```

---

# Brute-Force OpenSSL Encrypted GZIP

## Main Command

```bash
for i in $(cat rockyou.txt);do openssl enc -aes-256-cbc -d -in GZIP.gzip -k $i 2>/dev/null| tar xz;done
```

---

# Purpose

- Tests passwords from `rockyou.txt`
- Attempts decryption
- Extracts archive if password is correct

# Cracking BitLocker Encrypted Drives

BitLocker is Microsoft's full-disk encryption feature.

Features:
- AES-128 / AES-256 encryption
- Password or PIN protection
- Recovery key support

---

# Extract BitLocker Hashes

## Step 1: Generate Hashes

### Main Command

```bash
bitlocker2john -i Backup.vhd > backup.hashes
```

---

## Step 2: Extract Password Hash

```bash
grep "bitlocker\$0" backup.hashes > backup.hash
```

---

## Step 3: View Hash

```bash
cat backup.hash
```

---

# Crack BitLocker Password with Hashcat

Hashcat mode for BitLocker:
```text
22100
```

---

## Main Command

```bash
hashcat -a 0 -m 22100 <hash> /usr/share/wordlists/rockyou.txt
```

---

## Example Result

```text
1234qwer
```

---

# Important Notes About BitLocker

- Cracking can be very slow
- AES encryption is strong
- GPU acceleration greatly helps

---

# Mounting BitLocker Drives in Linux

Tool used:
```text
dislocker
```

---

# Install dislocker

## Main Command

```bash
sudo apt-get install dislocker
```

---

# Create Mount Directories

```bash
sudo mkdir -p /media/bitlocker
sudo mkdir -p /media/bitlockermount
```

---

# Configure Loop Device

```bash
sudo losetup -f -P Backup.vhd
```

---

# Decrypt Drive

```bash
sudo dislocker /dev/loop0p2 -u1234qwer -- /media/bitlocker
```

---

# Mount Decrypted Volume

```bash
sudo mount -o loop /media/bitlocker/dislocker-file /media/bitlockermount
```

---

# Browse Mounted Files

```bash
cd /media/bitlockermount/
ls -la
```

---

# Unmount BitLocker Drive

```bash
sudo umount /media/bitlockermount
sudo umount /media/bitlocker
```

---

# Key Points

- Archive files often contain sensitive data
- Use `2john` tools to extract hashes
- ZIP cracking process:
  1. Extract hash
  2. Crack password
- OpenSSL encrypted GZIP files may require brute-force loops
- BitLocker cracking requires:
  - `bitlocker2john`
  - Hashcat mode `22100`

---

# Important Notes

- Strong encryption can make cracking impractical
- GPU acceleration improves cracking performance
- Password reuse increases cracking success
- Custom wordlists are more effective than generic lists
