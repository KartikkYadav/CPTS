# 📌 Introduction to Hashcat - CPTS Notes

---

# 🔍 What is Hashcat

Hashcat is a powerful password cracking tool available for:
- Linux
- Windows
- macOS

It is one of the fastest password cracking tools because it supports **GPU acceleration**, making it much faster than CPU-based cracking in many situations.

Hashcat supports:
- Dictionary attacks
- Rule-based attacks
- Mask attacks
- Brute-force attacks

It is widely used in penetration testing and password auditing because of its speed and support for hundreds of hash types. :contentReference[oaicite:0]{index=0}  

---

# ⚙️ Basic Hashcat Syntax ⭐ IMPORTANT

## ⭐ MAIN COMMAND

```bash
hashcat -a 0 -m 0 <hashes> [wordlist, rule, mask]
```

---

# 📌 Important Arguments

| Argument | Meaning |
|----------|----------|
| `-a` | Attack mode |
| `-m` | Hash type |
| `<hashes>` | Hash or hash file |
| `wordlist/rule/mask` | Attack input |

---

# 🔍 Hash Types ⭐ VERY IMPORTANT

Hashcat supports hundreds of hash formats.

Each hash type has a numeric ID.

---

# ⭐ MAIN COMMAND

```bash
hashcat --help
```

Used to display:
- Supported hash types
- Attack modes
- Options

---

# 📌 Common Hash Modes

| Hash Type | Mode |
|-----------|------|
| MD5 | 0 |
| SHA1 | 100 |
| SHA256 | 1400 |
| SHA512 | 1700 |
| NTLM | 1000 |

---

# 🔍 Identifying Hash Types ⭐ IMPORTANT

Sometimes the hash type is unknown.

We can use `hashid` to identify it.

---

## ⭐ MAIN COMMAND

```bash
hashid -m <hash>
```

---

## 📌 Example

```bash
hashid -m '$1$FNr44XZC$wQxY6HHLrgrGX0e1195k.1'
```

---

## 📌 Output Example

```text
MD5 Crypt [Hashcat Mode: 500]
```

This tells us:
- Hash type = MD5 Crypt
- Hashcat mode = 500

---

# ⚡ Attack Modes ⭐ IMPORTANT

Hashcat supports multiple attack modes.

Most important:
- Dictionary Attack
- Mask Attack

---

# 1️⃣ Dictionary Attack ⭐ VERY IMPORTANT

## 🔍 Definition
Uses a wordlist to test common passwords against hashes.

Most practical attack method in real-world pentesting.

---

## ⭐ MAIN COMMAND

```bash
hashcat -a 0 -m 0 <hash> <wordlist>
```

---

## 📌 Example

```bash
hashcat -a 0 -m 0 e3e3ec5831ad5e7288241960e5d4fdb8 /usr/share/wordlists/rockyou.txt
```

---

# 📚 Popular Wordlists ⭐ IMPORTANT

## rockyou.txt

Location:
```bash
/usr/share/wordlists/rockyou.txt
```

Contains:
- Millions of leaked passwords

---

# 🔧 Rules in Hashcat ⭐ IMPORTANT

Rules modify passwords from a wordlist to generate more password guesses.

Examples:
```text
password → Password1
admin → Admin123!
```

---

# 📂 Rule Files Location ⭐ IMPORTANT

```bash
/usr/share/hashcat/rules
```

---

# ⭐ View Rules

```bash
ls -l /usr/share/hashcat/rules
```

---

# 📌 Popular Rule Files

| Rule | Purpose |
|------|----------|
| best64.rule | Common password modifications |
| leetspeak.rule | Leetspeak replacements |
| rockyou-30000.rule | Large rule set |

---

# ⭐ RULE-BASED ATTACK COMMAND

```bash
hashcat -a 0 -m 0 <hash> rockyou.txt -r /usr/share/hashcat/rules/best64.rule
```

---

## 📌 Why Rules Matter
Many users:
- Add numbers
- Replace letters
- Add symbols

Rules simulate these common behaviors.

---

# 2️⃣ Mask Attack ⭐ VERY IMPORTANT

## 🔍 Definition
Mask attack is a targeted brute-force attack.

Instead of testing every possible password combination, we define a pattern (mask).

This greatly reduces attack time.

---

# 📌 Built-in Character Sets ⭐ IMPORTANT

| Symbol | Meaning |
|--------|----------|
| `?l` | lowercase letters |
| `?u` | uppercase letters |
| `?d` | digits |
| `?s` | symbols |
| `?a` | all characters |

---

# 📌 Example Pattern

Password structure:
```text
1 uppercase + 4 lowercase + 1 digit + 1 symbol
```

Mask:
```text
?u?l?l?l?l?d?s
```

---

# ⭐ MAIN COMMAND

```bash
hashcat -a 3 -m 0 <hash> '?u?l?l?l?l?d?s'
```

---

## 📌 Example

```bash
hashcat -a 3 -m 0 1e293d6912d074c0fd15844d803400dd '?u?l?l?l?l?d?s'
```

---

# 🔧 Custom Character Sets ⭐ IMPORTANT

Can define custom sets using:
- `-1`
- `-2`
- `-3`
- `-4`

Referenced as:
```text
?1 ?2 ?3 ?4
```

---

# ⚡ CPTS Key Points ⭐

- Hashcat is GPU accelerated
- `-m` specifies hash type
- `-a` specifies attack mode
- Dictionary attacks are most practical
- Rules improve cracking success

---

# ⚠️ Important Notes

- GPU greatly improves cracking speed
- Strong passwords take much longer
- Wrong hash mode = failed cracking
- Rule-based attacks are very effective
