# Introduction to Web Shells

## Overview

Modern organizations heavily rely on:
- Web applications
- Cloud platforms
- Browser-based services

Because of this, web applications are one of the most common attack surfaces during:
- External penetration tests
- Bug bounty engagements
- Red team operations

# Common Initial Access Methods

Attackers often gain footholds through:
- File upload vulnerabilities
- SQL Injection
- Command Injection
- LFI/RFI
- Password spraying
- Misconfigured services
- Social engineering

---

# Why Web Applications Matter

Organizations often secure:
- SMB
- RDP
- Internal services

But still expose:
- Websites
- Login portals
- APIs
- Web applications

This makes web attacks extremely common.

---

# What is a Web Shell?

A web shell is:
```text
A browser-based shell used to execute commands on a web server
```

It allows attackers to:
- Execute system commands
- Browse files
- Upload/download files
- Gain remote code execution

through the browser.

---

# How Web Shells Are Obtained

Usually by exploiting:
- Unrestricted file uploads
- Vulnerable applications
- Misconfigured FTP services
- WAR/JSP deployments
- Weak upload validation

---

# Common Upload Targets

Attackers commonly upload web shells through:
- Profile picture uploads
- File upload forms
- Admin panels
- CMS plugins
- Web application features

# Example Scenario

Attacker uploads:

```text
shell.php
```

Then accesses:

```text
http://target/shell.php
```

This may provide:
- Command execution
- Reverse shell access
- File management

---

# Important Observation

Web shells are often:
- Unstable
- Temporary
- Automatically deleted

Because of this, attackers usually:
1. Gain initial access via web shell
2. Upgrade to reverse shell
3. Establish persistence

---

# Why Reverse Shells Are Preferred

Reverse shells provide:
- Better interaction
- Stable TTY
- Full command execution
- Easier privilege escalation

---

# Key Points

- Web applications are major attack surfaces
- File upload vulnerabilities commonly lead to RCE
- Web shells provide browser-based command execution
- PHP, JSP, and ASPX shells are common
- Web shells are often used for initial access only

---

# One-Line Revision

A web shell is a browser-accessible payload uploaded through a vulnerable web application to achieve remote command execution on the web server.
