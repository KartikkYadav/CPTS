## Overview

Over 70% of web servers run on Unix/Linux systems, making Linux exploitation an important skill for penetration testers.

Attackers commonly target:
- Web applications
- Services
- Misconfigurations
- Vulnerable software

# Problem: Non-TTY Shell

Sometimes the shell:
- Has limited functionality
- Lacks proper prompt
- Cannot use commands like:
  - `su`
  - `sudo`

This usually happens when running as:
```text
apache
```

or another service account.

# Purpose of TTY Shell

Improves shell functionality by enabling:
- Interactive commands
- Better terminal handling
- Privilege escalation attempts
- `su` and `sudo`

# Key Points

- Linux web servers are common targets
- Nmap helps identify services and technologies
- Web applications are common entry points
- Metasploit automates exploitation
- Non-TTY shells have limited functionality
- Python can spawn interactive TTY shells

# One-Line Revision

Linux infiltration commonly involves enumerating services, exploiting vulnerable web applications, gaining Meterpreter access, and upgrading non-interactive shells into full TTY sessions.
