# Spawning Interactive Shells

## Overview

Sometimes after exploitation we only get:
- Limited shells
- Non-interactive shells
- Jail shells

These shells may:
- Lack a proper prompt
- Prevent `sudo` usage
- Prevent job control
- Break interactive commands

To improve shell functionality, attackers spawn:
```text
Interactive TTY shells
```

# /bin/sh Interactive Shell

## Main Command

```bash
/bin/sh -i
```

---

# Purpose

Runs:
```text
/bin/sh
```

in:
```text
interactive mode
```

---

# Example Output

```text
sh: no job control in this shell
sh-4.2$
```

---

# Python TTY Shell

## Main Command

```bash
python -c 'import pty; pty.spawn("/bin/sh")'
```

---

# Purpose

- Spawns proper TTY shell
- Improves shell stability
- Enables interactive commands

Very commonly used during:
- Linux privilege escalation
- Post-exploitation

---

# Perl Shell

## Main Command

```bash
perl -e 'exec "/bin/sh";'
```

---

# Purpose

Uses Perl to execute:
```text
/bin/sh
```

Useful when:
- Python is unavailable
- Perl exists on system

---

# Ruby Shell

## Main Command

```bash
ruby -e 'exec "/bin/sh"'
```

---

# Purpose

Uses Ruby interpreter to spawn shell.

---

# Lua Shell

## Main Command

```bash
lua -e 'os.execute("/bin/sh")'
```

---

# Purpose

Executes shell using:
```text
os.execute()
```

---

# AWK Shell

## Main Command

```bash
awk 'BEGIN {system("/bin/sh")}'
```

---

# Purpose

Uses AWK's:
```text
system()
```

function to execute shell.

---

# Find Command Shell

## Main Command

```bash
find . -exec /bin/sh \; -quit
```

---

# Purpose

Uses:
```text
-exec
```

to directly launch shell.

---

# Alternative Find + AWK Method

## Main Command

```bash
find / -name file -exec /bin/awk 'BEGIN {system("/bin/sh")}' \;
```

---

# Vim Shell Escape

## Direct Command

```bash
vim -c ':!/bin/sh'
```

---

# Interactive Vim Escape

## Main Commands

```vim
:set shell=/bin/sh
:shell
```

---

# Purpose

Escapes from Vim into system shell.

Useful when:
- Vim allowed via sudo
- Restricted shell environments

# Checking File Permissions

Before executing binaries or scripts:
- Check permissions

---

## Main Command

```bash
ls -la <file>
```

---

# Checking Sudo Permissions

## Main Command

```bash
sudo -l
```

---

# Example Output

```text
(ALL : ALL) NOPASSWD: ALL
```

---

# Important Meaning

The current user can run:
```text
ALL commands
```

as:
```text
root
```

without password.

This usually leads to:
```text
Privilege Escalation
```

---

# Important Notes

- Non-TTY shells have limited functionality
- Interactive shells improve command execution
- Python TTY is most common
- Perl/Ruby/Lua useful fallback options
- `sudo -l` often reveals escalation paths

---

# Key Points

- Limited shells can be upgraded
- Multiple scripting languages can spawn shells
- Vim can escape to system shell
- File permissions matter during exploitation
- Stable TTY required for many commands

---

# One-Line Revision

Interactive shell spawning upgrades limited shells into stable TTY sessions using interpreters like Python, Perl, Ruby, AWK, Lua, Vim, and shell binaries.
