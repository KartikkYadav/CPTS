# PHP Web Shells

## Overview

PHP (Hypertext Preprocessor) is a popular:
- Server-side scripting language
- Web development language

It is heavily used in:
- Websites
- CMS platforms
- Web applications

Because of its popularity, PHP applications are common pentest targets.

---

# PHP in Web Applications

PHP processes:
- User input
- Forms
- Authentication
- Backend logic

Example:

```text
login.php
```

processes:
- Username
- Password
- Login validation

---

# Why PHP Matters for Pentesters

If a server uses PHP, attackers may:
- Upload PHP payloads
- Execute commands
- Gain web shell access
- Obtain reverse shells

---

# Example Target

The rConfig application:
- Uses PHP
- Runs on Linux
- Contains file upload functionality

Default credentials:

```text
admin:admin
```

---

# Uploading a PHP Web Shell

Attackers used:
```text
Vendor Logo Upload
```

feature to upload:
```text
connect.php
```

---

# Problem

The application only allowed:
- `.png`
- `.jpg`
- `.gif`

uploads.

---

# Bypassing File Type Restrictions

Burp Suite was used to intercept the upload request.

---

# Configure Burp Proxy

## Proxy Settings

```text
127.0.0.1:8080
```

---

# Important Modification

Change:

```text
Content-Type: application/x-php
```

to:

```text
Content-Type: image/gif
```

---

# Purpose

This tricks the application into accepting:
```text
.php
```

file as an image.

---

# Successful Upload

After forwarding the request:
- Vendor added successfully
- PHP shell uploaded

---

# Accessing the Web Shell

## Example Path

```text
/images/vendor/connect.php
```

---

# Result

Executing the PHP file through browser provides:
- Command execution
- Browser-based shell
- Access to underlying Linux OS

---

# Example Capabilities

- Run system commands
- Read files
- Enumerate system
- Attempt privilege escalation

---

# Important Observation

The shell was:
```text
Non-interactive
```

Meaning:
- Limited command chaining
- Reduced shell functionality
- Less stable than reverse shells

---

# Common Issues with Web Shells

| Problem | Description |
|---------|-------------|
| File deletion | Uploads removed automatically |
| Limited interaction | Non-TTY shell |
| Logging | Web requests logged |
| Detection | AV/WAF detection |
| Instability | Commands may fail |

---

# Important Notes

- Reverse shells are usually preferred
- Delete payloads after gaining access
- Document:
  - File names
  - Upload locations
  - Hashes
  - Exploitation steps

---

# OPSEC Considerations

Good practice:
- Remove comments from payloads
- Rename files
- Delete shells after use
- Avoid leaving artifacts behind

---

# Key Points

- PHP web shells target Linux web servers
- Burp Suite can bypass upload restrictions
- Content-Type manipulation is common
- Web shells often provide initial access only
- Reverse shells provide better stability

---

# One-Line Revision

PHP web shells allow attackers to execute commands on Linux web servers by uploading malicious `.php` files through vulnerable file upload functionality.
