# Anatomy of a Shell

## Overview

Every operating system uses a shell to interact with the system.

To access the shell, we use a:
```text
Terminal Emulator
```

The terminal emulator provides the interface, while the shell interprets commands.

# Command Language Interpreter

A shell is also called a:
- Command Language Interpreter
- Command and Scripting Interpreter

Its job is to:
- Interpret user commands
- Send instructions to the operating system

Examples:
- Bash
- PowerShell
- Zsh
- Ksh

---

# Shell Prompt

In Linux shells like Bash, the prompt usually contains:

```text
$
```

Example:

```bash
karthikk007@htb[/htb]$
```

This indicates the shell is ready to receive commands.

---

# Invalid Command Example

Typing an unknown command:

```bash
meep
```

Output:

```text
command not found
```

This message comes from the shell interpreter.

---

# Identify Current Shell

## Using ps

```bash
ps
```

Example Output:

```text
PID TTY          TIME CMD
4232 pts/1    00:00:00 bash
```

Shows:
```text
bash
```

is the current shell.

---

# Using Environment Variables

## Main Command

```bash
env
```

Example Output:

```text
SHELL=/bin/bash
```

---

# Bash vs PowerShell

The same terminal emulator can run different shells.

Examples:
- Bash
- PowerShell

Both:
- Accept commands
- Execute scripts
- Interact with the OS

But they use:
- Different syntax
- Different commands
- Different scripting methods

---

# Important Observation

A terminal emulator:
- Is NOT the shell itself

The shell can be changed depending on:
- User preference
- Workflow
- Operating system

---

# Key Points

- Terminal emulator provides the interface
- Shell interprets commands
- Bash commonly uses `$`
- `ps` and `env` help identify the current shell
- Different shells support different commands and scripting languages

---

# One-Line Revision

A shell is a command interpreter accessed through a terminal emulator, allowing users to interact with the operating system using commands and scripts.
