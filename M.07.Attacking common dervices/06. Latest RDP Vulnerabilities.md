# Latest RDP Vulnerabilities

---

In 2019, a critical vulnerability was published in the RDP (TCP/3389) service that led to remote code execution (RCE) with the identifier **CVE-2019-0708**. This vulnerability is widely known as **BlueKeep**.

* **Pre-requisites:** It does not require prior access or valid credentials to exploit the target system.
* **Impact:** The exploitation of this vulnerability frequently leads to severe malware or ransomware deployment.
* **Target Vulnerability:** Large organizations such as hospitals, whose software infrastructure is explicitly locked to legacy versions and dependencies, are particularly vulnerable because infrastructure maintenance and updates are highly costly.

---

## The Concept of the Attack

Similar to certain SMB exploits, this vulnerability is based on manipulated network requests sent to the targeted service. The critical risk factor is that **no user authentication is required** to trigger the flaw. Instead, the vulnerability is executed right after initializing the connection when basic environmental settings are exchanged between the client and the server.

This memory corruption flaw follows a **Use-After-Free (UAF)** technique:

> **Use-After-Free (UAF):** A vulnerability condition that occurs when a program continues to use a pointer after the memory allocation it references has been freed. Attackers can abuse this by filling the freed memory block with malicious instructions, which the CPU then unexpectedly executes.

This attack involves highly complex kernel-level interactions. After the vulnerable function is targeted and the allocated memory is freed, specialized data is written directly to the kernel space. This allows the attacker to overwrite the kernel memory, place custom execution instructions inside the freed memory allocation, and force the CPU to execute them.

---

## Initiation of the Attack

| Step | BlueKeep Concept of Attacks | Category |
| --- | --- | --- |
| **1.** | The source is the initialization request of the settings exchange between server and client that the attacker has manipulated. | **Source** |
| **2.** | The request leads to a internal function used to create a virtual channel containing the vulnerability. | **Process** |
| **3.** | Since this service is suitable for administering of the system, it is automatically run with the **LocalSystem Account** privileges of the system. | **Privileges** |
| **4.** | The manipulation of the function redirects execution to a kernel process. | **Destination** |

This concludes the initial exploitation phase. The execution cycle then starts over, specifically focused on establishing remote code execution.

---

## Trigger Remote Code Execution

| Step | BlueKeep Concept of Attacks | Category |
| --- | --- | --- |
| **5.** | The source this time is the custom payload created by the attacker that is inserted into the process to free the memory in the kernel and place our instructions. | **Source** |
| **6.** | The process in the kernel is triggered to free the kernel memory and force the CPU instruction pointer to point to our shellcode. | **Process** |
| **7.** | Since the Windows Kernel runs with the highest possible system level ring privileges, the instructions placed into the freed kernel memory are also executed with **LocalSystem Account** privileges. | **Privileges** |
| **8.** | With the execution of our instructions directly from the kernel space, a reverse shell is established and sent over the network back to our attack host. | **Destination** |

## Current Landscape & CPTS Exam Operational Notes

According to Microsoft, not all newer Windows variants are vulnerable to BlueKeep. While security updates for supported current Windows versions are available, Microsoft went as far as providing legacy updates for many older Windows versions that had already passed their official end-of-support lifecycle.

Despite these patches, an initial internet-wide scan in May 2019 identified over **950,000** Windows systems exposed and vulnerable to BlueKeep attacks. Even today, a large number of these hosts remain unpatched.

---

### ⚠️ Critical Operational Warning for Penetration Testers

While BlueKeep is a severe vulnerability that you will likely encounter during real-world network penetration tests and assessments, you must exercise extreme caution before running a public exploit against it:

* **System Instability:** The exploitation process modifies memory deep within the Windows Kernel. A minor timing error, network packet drop, or memory alignment issue will immediately crash the entire target server, causing a **Blue Screen of Death (BSoD)**.
* **Standard Operating Procedure:** Never launch a BlueKeep exploit on a production network without explicit, written authorization.
* **Client Advisory:** If you identify a vulnerable host, it is best practice to speak directly with your client first. Ensure they understand the exact operational downtime risks, and let them explicitly decide whether they want you to execute the live exploit or simply document the vulnerability based on version enumeration.
