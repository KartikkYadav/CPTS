# Directory Fuzzing with FFUF

## Overview

Directory fuzzing is used to discover:
- Hidden directories
- Web pages
- Admin panels
- Backup files
- Sensitive endpoints

This is done by:
- Sending many HTTP requests
- Using a wordlist
- Checking server responses

FFUF is one of the fastest and most commonly used web fuzzing tools.

---

# FFUF

## Overview

FFUF (`Fuzz Faster U Fool`) is a fast web fuzzing tool used for:
- Directory discovery
- File discovery
- Parameter fuzzing
- Virtual host fuzzing

Pre-installed on:
- Kali Linux
- Parrot OS
- Pwnbox

---

# Install FFUF

## Main Command

```bash
sudo apt install ffuf -y
```

---

# Help Menu

## Main Command

```bash
ffuf -h
```

---

# Important FFUF Options

| Option | Purpose |
|--------|----------|
| `-w` | Wordlist |
| `-u` | Target URL |
| `-mc` | Match status codes |
| `-fc` | Filter status codes |
| `-fs` | Filter response size |
| `-t` | Threads |
| `-o` | Save output |
| `-H` | Add headers |
| `-X` | HTTP method |
| `-d` | POST data |

---

# FUZZ Keyword

FFUF uses:
```text
FUZZ
```

as placeholder for payload insertion.

---

# Basic Directory Fuzzing

## Main Command

```bash
ffuf -w /opt/useful/seclists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://SERVER_IP:PORT/FUZZ
```

---

# Command Breakdown

| Part | Purpose |
|------|----------|
| `-w` | Wordlist path |
| `FUZZ` | Keyword placeholder |
| `-u` | Target URL |

---

# Example Result

```text
blog [Status: 301]
```

---

# Important HTTP Status Codes

| Status | Meaning |
|--------|----------|
| 200 | OK |
| 301/302 | Redirect |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not Found |

---

# Default Matchers

FFUF matches:
```text
200,204,301,302,307,401,403
```

by default.

---

# Filter Responses

## Filter by Status Code

```bash
-fc 404
```

Filters:
```text
404 Not Found
```

responses.

---

# Match All Responses

## Main Command

```bash
-mc all
```

---

# Filter by Response Size

## Main Command

```bash
-fs 42
```

Useful for:
- Removing false positives
- Ignoring default pages

---

# Recursive Fuzzing

## Main Command

```bash
-recursion
```

Allows:
- Recursive directory enumeration

---

# Increase Threads

## Main Command

```bash
-t 200
```

---

# Important Note

Higher thread count:
- Faster scanning
- Higher resource usage
- May cause:
  - DoS
  - Rate limiting
  - Detection

Recommended to avoid excessive threads on production systems.

---

# Example Full Scan

## Main Command

```bash
ffuf -w /opt/useful/seclists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://SERVER_IP:PORT/FUZZ -mc all -fs 42 -c -v
```

# Common Discoveries

Directory fuzzing may reveal:
- `/admin`
- `/backup`
- `/uploads`
- `/dev`
- `/api`
- `/config`

---

# Important Observation

A blank page does NOT mean:
```text
Directory is empty
```

There may still be:
- Hidden files
- APIs
- Uploads
- Sensitive endpoints

---

# Key Points

- FFUF is a fast web fuzzing tool
- Uses `FUZZ` keyword for injection
- Commonly used for directory enumeration
- Response filtering reduces false positives
- Large thread counts increase speed but also risk

---

# One-Line Revision

FFUF is a fast web fuzzing tool used to discover hidden directories, files, and endpoints by sending wordlist-based HTTP requests to a target web application.
