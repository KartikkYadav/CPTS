
# Page Fuzzing with FFUF

## Overview

After discovering hidden directories during directory fuzzing, the next step is to identify whether those directories contain hidden files or pages. Many times a directory may look completely empty when opened in a browser, but internally it may still contain:
- Login panels
- Admin pages
- APIs
- Backup files
- Sensitive scripts

This process of searching for hidden files and pages is known as:
```text
Page Fuzzing
```

FFUF is commonly used for this because it is extremely fast and supports fuzzing:
- Directories
- Files
- Extensions
- Parameters
- Virtual hosts

---

# Why Page Fuzzing is Important

Suppose we discovered:

```text
/blog
```

When we visit it manually:
- The page looks blank
- No links are visible
- No content is shown

This does NOT mean the directory is empty.

Many applications contain hidden pages such as:
- `admin.php`
- `login.php`
- `dashboard.php`
- `config.php`

that are not linked publicly.

Instead of manually guessing filenames, FFUF automates the process using large wordlists.

---

# Step 1 - Identify the Website Extension

Before fuzzing for files, we first need to determine:
```text
Which extension the website uses
```

Examples:
- PHP websites use `.php`
- ASP.NET uses `.aspx`
- Java applications may use `.jsp`

This is important because fuzzing:
```text
admin.php
```

on an ASP.NET server would fail.

---

# Extension Fuzzing

To identify the correct extension, we fuzz common extensions against a known filename.

The most common filename used is:
```text
index
```

because almost every website contains:
- `index.php`
- `index.aspx`
- `index.jsp`

---

# Extension Wordlist

FFUF uses the SecLists extension wordlist:

```text
/opt/useful/seclists/Discovery/Web-Content/web-extensions.txt
```

---

# Extension Fuzzing Command

```bash
ffuf -w /opt/useful/seclists/Discovery/Web-Content/web-extensions.txt:FUZZ -u http://SERVER_IP:PORT/blog/indexFUZZ
```

---

# Understanding the Command

Here:
- `-w` specifies the wordlist
- `FUZZ` acts as placeholder
- FFUF replaces `FUZZ` with each extension from the wordlist

So requests become:

```text
/blog/index.php
/blog/index.aspx
/blog/index.jsp
/blog/index.html
```

and many more.

---

# Example Output

```text
.php   [Status: 200]
.phps  [Status: 403]
```

---

# Understanding the Results

A response with:
```text
Status 200
```

means:
```text
The file exists and is accessible
```

A response with:
```text
403 Forbidden
```

usually means:
```text
The file exists but access is denied
```

From this result we conclude:
```text
The website is running PHP
```

because:
```text
index.php
```

returned a successful response.

---

# Step 2 - Page Fuzzing

Now that we know the extension is:
```text
.php
```

we can fuzz for hidden PHP pages.

---

# Page Fuzzing Command

```bash
ffuf -w /opt/useful/seclists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://SERVER_IP:PORT/blog/FUZZ.php
```

---

# What This Command Does

FFUF now replaces:
```text
FUZZ
```

with words from the wordlist.

Examples:

```text
/blog/admin.php
/blog/login.php
/blog/dashboard.php
/blog/config.php
```

---

# Example Results

```text
index      [Status: 200, Size: 0]
admin      [Status: 200, Size: 465]
```

---

# Understanding the Results

The:
```text
index.php
```

page returned:
```text
Size: 0
```

which usually indicates:
- Blank page
- Empty response
- No visible content

However:
```text
admin.php
```

returned:
```text
Size: 465
```

which strongly suggests:
- Real content exists
- The page contains data
- Possibly an admin panel

---

# Important FFUF Response Fields

| Field | Meaning |
|------|----------|
| Status | HTTP response code |
| Size | Response size in bytes |
| Words | Number of words |
| Lines | Number of lines |

These help distinguish:
- Real pages
- False positives
- Empty responses

---

# Important HTTP Status Codes

| Code | Meaning |
|------|----------|
| 200 | Accessible |
| 301/302 | Redirect |
| 401 | Authentication required |
| 403 | Forbidden |
| 404 | Not Found |

---

# Filtering False Positives

Some applications return:
```text
Fake 200 responses
```

for non-existing pages.

In these cases:
- Response size analysis becomes important
- Filtering improves accuracy

---

# Filter 404 Responses

```bash
-fc 404
```

This removes:
```text
404 Not Found
```

responses.

---

# Filter by Response Size

```bash
-fs 0
```

Useful for:
- Ignoring empty pages
- Removing false positives

---

# Match All Responses

```bash
-mc all
```

Tells FFUF to:
```text
Match every HTTP status code
```

---

# Common Hidden Pages Found During Fuzzing

Page fuzzing commonly reveals:
- Admin panels
- Login pages
- Backup files
- Development pages
- API endpoints
- Old application versions

Examples:

```text
/admin.php
/dev.php
/backup.php
/test.php
/api.php
```

---

# Common Wordlists

| Wordlist | Purpose |
|----------|----------|
| `directory-list-2.3-small.txt` | Page discovery |
| `web-extensions.txt` | Extension discovery |
| `raft-*` | Large fuzzing lists |

---

# Key Points

Page fuzzing is used after directory discovery to identify hidden files inside directories. The process usually starts by identifying the web technology and extension being used, followed by fuzzing filenames using FFUF and large wordlists. Response size and HTTP status codes help determine which pages are real and worth investigating further.

---

# One-Line Revision

Page fuzzing with FFUF discovers hidden files and web pages by first identifying the correct extension and then fuzzing filenames inside discovered directories using wordlists.
