# Recursive Fuzzing with FFUF

## Overview

During normal directory fuzzing, we usually follow a manual workflow:
1. Discover a directory
2. Enter that directory
3. Start another fuzzing scan inside it
4. Repeat the process again and again

This works fine for very small applications, but real-world web applications are usually much larger and contain deeply nested directory structures.

For example, a website may contain paths like:

```text
/admin/users/uploads/images/
```

or

```text
/forum/members/profile/settings/
```

Manually fuzzing every single directory one by one would become:
- Extremely time consuming
- Repetitive
- Inefficient

To solve this problem, FFUF supports:
```text
Recursive Fuzzing
```

which automates the process of entering newly discovered directories and fuzzing inside them automatically.

---

# What is Recursive Fuzzing?

Recursive fuzzing means:
```text
FFUF automatically continues scanning inside directories it discovers
```

Instead of manually running multiple scans, FFUF creates:
```text
A scanning tree
```

As soon as a new directory is found, FFUF starts another fuzzing job inside that directory.

For example:

First FFUF discovers:

```text
/blog
```

Then automatically starts fuzzing:

```text
/blog/FUZZ
```

If it later discovers:

```text
/blog/admin
```

it automatically continues scanning:

```text
/blog/admin/FUZZ
```

This allows very fast and automated discovery of:
- Hidden pages
- Subdirectories
- APIs
- Admin panels
- Upload folders

across an entire application.

---

# Why Recursive Fuzzing is Important

Modern web applications are usually very large and may contain:
- Thousands of directories
- Deep folder structures
- Hidden development environments
- Multiple APIs
- Backup locations

Without recursive fuzzing, attackers or pentesters would need to:
- Re-run FFUF manually
- Track discovered directories
- Start scans repeatedly

Recursive fuzzing automates all of this.

This makes reconnaissance:
- Faster
- More efficient
- More complete

especially during:
- CPTS exams
- Bug bounty hunting
- Web pentesting
- Internal assessments

---

# The Main Problem with Recursive Scanning

Recursive scanning is powerful, but it also introduces a problem:
```text
The scan size grows very quickly
```

Every newly discovered directory creates:
```text
Another scan job
```

A website with many directories may suddenly generate:
- Hundreds of thousands of requests
- Millions of requests
- Extremely long scans

For example:

```text
/login/user/uploads/images/files/
```

contains multiple nested directories.

If recursion is unrestricted, FFUF may continue scanning forever until every possible subdirectory is tested.

This can:
- Slow down your system
- Trigger WAFs
- Cause rate limiting
- Generate alerts
- Impact the target server

---

# Recursion Depth

To control this problem, FFUF allows limiting:
```text
How deep the scan goes
```

using:

```bash
-recursion-depth
```

---

# Understanding Recursion Depth

## Depth 1

```bash
-recursion-depth 1
```

This tells FFUF:
- Scan the main directory
- Scan one level below it

Example:

FFUF scans:

```text
/blog/
/forum/
/admin/
```

and their direct contents.

But it will NOT continue deeper into:

```text
/blog/uploads/
/forum/users/
```

---

## Depth 2

```bash
-recursion-depth 2
```

Now FFUF scans:
- Main directories
- Subdirectories
- Sub-subdirectories

The deeper the recursion:
- The larger the scan
- The slower the scan
- The more requests generated

---

# Enabling Recursive Scanning

Recursive fuzzing is enabled using:

```bash
-recursion
```

Without this flag:
- FFUF only scans the current path
- No automatic directory traversal happens

---

# Fuzzing Extensions During Recursion

While recursively scanning directories, FFUF can also fuzz:
```text
File extensions
```

using:

```bash
-e .php
```

This means FFUF will test:
- Directories
- PHP pages

at the same time.

---

# Why Extension Fuzzing Helps

Most websites use:
```text
A consistent technology stack
```

If the website uses PHP, then most pages likely end with:
```text
.php
```

Instead of manually testing:
- `/admin`
- `/admin.php`

FFUF automatically tests both.

This improves:
- Coverage
- Efficiency
- Discovery rate

---

# Verbose Output

Recursive scans often discover many files named:
```text
index.php
```

Without full URLs, it becomes difficult to know:
```text
Which directory the file belongs to
```

To solve this, we use:

```bash
-v
```

which enables:
```text
Verbose output
```

and shows full URLs.

---

# Recursive Fuzzing Command

```bash
ffuf -w /opt/useful/seclists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://SERVER_IP:PORT/FUZZ -recursion -recursion-depth 1 -e .php -v
```

---

# Understanding the Command

This command tells FFUF to:
1. Use the directory wordlist
2. Fuzz the target URL
3. Automatically recurse into directories
4. Only scan one level deep
5. Test PHP pages
6. Display full URLs

---

# Example Output

```text
[INFO] Adding a new job to the queue: http://SERVER_IP:PORT/forum/FUZZ
```

This means:
```text
FFUF discovered /forum and automatically started another scan inside it
```

---

# Example Discoveries

```text
/blog/index.php
/forum/index.php
```

This shows that FFUF:
- Entered the discovered directories
- Continued fuzzing automatically
- Found additional files

without requiring manual interaction.

# Why Recursive Scans Become Large

In the example:
- The scan took much longer
- Requests increased heavily
- The wordlist was effectively tested multiple times

This happens because FFUF now fuzzes:
- Directories
- PHP files
- Subdirectories

all together.

Recursive fuzzing trades:
```text
More requests
```

for:
```text
Better coverage and automation
```

# Operational Security Considerations

Aggressive recursive scans can easily:
- Trigger IDS/WAF alerts
- Cause IP bans
- Generate excessive logs
- Stress production systems

During real engagements:
- Limit recursion depth
- Reduce threads if needed
- Avoid unnecessary noise

Always scan responsibly.

---

# Key Points

Recursive fuzzing automates the process of scanning discovered directories and subdirectories, making reconnaissance much faster and more scalable on large applications. However, because recursive scans generate many requests very quickly, recursion depth and proper scan control become extremely important to avoid noisy or excessive scans.

# One-Line Revision

Recursive fuzzing in FFUF automatically scans newly discovered subdirectories and files, allowing large-scale automated web enumeration with controlled recursion depth.
