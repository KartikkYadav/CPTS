# Solve

# Sub-Domain Fuzzing

In the SecLists repo, there is a specific section for sub-domain wordlists, consisting of common words usually used for sub-domains.

We can find it in:

```text
/SecLists/Discovery/DNS/
```

In our case, we would be using a shorter wordlist:

```text
subdomains-top1million-5000.txt
```

## Command

```bash
ffuf -w /SecLists/Discovery/DNS/subdomains-top1million-5000.txt:FUZZ -u http://FUZZ.academy.htb:PORT/
```

### Explanation

| Option | Meaning |
|--------|----------|
| `-u` | Target URL |
| `-w` | Wordlist file path |

As we can see, there are no results available.

This just means there are no PUBLIC sub-domains, as there is no public sub-domain DNS record.

And this prompts us to try:
```text
VHosts fuzzing
```

---

# VHosts Fuzzing

## Command

```bash
ffuf -w Seclists/Discovery/DNS/subdomains-top1million-5000.txt:FUZZ -u http://academy.htb:PORT/ -H "Host: FUZZ.academy.htb"
```

### Explanation

| Option | Meaning |
|--------|----------|
| `-u` | Target URL |
| `-w` | Wordlist file path |
| `-H` | Header "Name: Value" |

Most of our results will register with:
```text
200 OK
```

So we need another way to filter the output.

---

# Filtering Results

We can use:

```bash
-fs
```

which means:
```text
Filter HTTP response size
```

## Command

```bash
ffuf -w Seclists/Discovery/DNS/subdomains-top1million-5000.txt:FUZZ -u http://academy.htb:PORT/ -H "Host: FUZZ.academy.htb" -fs 985
```

Great! Here we have obtained the answer.

## Answer

```text
test
archive
faculty
```

# Question 3

## One of the pages you will identify should say:

```text
You don’t have access!
```

What is the full page URL?

---

# Solve

At this stage, we need:
```text
Recursive page fuzzing
```

because:
- We do not know the exact location
- The page may exist inside hidden directories

Recursive fuzzing automatically searches:
- Directories
- Subdirectories
- Files

---

# Recursive Fuzzing Command

```bash
ffuf -w SecLists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://academy.htb:PORT/FUZZ -recursion -recursion-depth 1 -e .php,.php7,.phps
```

We must filter the results using:

```bash
-fs
```

---

# Filtered Recursive Scan

```bash
ffuf -w SecLists/Discovery/Web-Content/directory-list-2.3-small.txt:FUZZ -u http://academy.htb:PORT/FUZZ -recursion -recursion-depth 1 -e .php,.php7,.phps -fs 287
```

Perform the same process for:
```text
test
archive
faculty
```

Eventually, an interesting result appears under:

```text
faculty.academy.htb
```

The recursive fuzzing identifies:

```text
faculty.academy.htb/courses/
```

Inside this directory, we discover:

```text
linux-security.php7
```

The page size:
```text
774
```

indicates real content.

# Parameter Fuzzing Command

```bash
ffuf -w SecLists/Discovery/Web-Content/burp-parameter-names.txt:FUZZ -u http://faculty.academy.htb:PORT/courses/linux-security.php7 -X POST -d 'FUZZ=key' -H 'Content-Type: application/x-www-form-urlencoded'
```

### Explanation

| Option | Meaning |
|--------|----------|
| `-X` | HTTP method |
| `-d` | POST data |
| `-H` | Add Header |

---

# Filtering the Results

We filter by size:


## Final Command

```bash
ffuf -w SecLists/Discovery/Web-Content/burp-parameter-names.txt:FUZZ -u http://faculty.academy.htb:PORT/courses/linux-security.php7 -X POST -d 'FUZZ=key' -H 'Content-Type: application/x-www-form-urlencoded' -fs 774
```

# Key Learning Points

- Subdomain fuzzing discovers public DNS records
- VHost fuzzing discovers hidden virtual hosts
- Recursive fuzzing discovers hidden pages and directories
- Extension fuzzing identifies backend technologies
- Parameter fuzzing discovers hidden GET/POST parameters
- Value fuzzing discovers valid parameter values
- Filtering with `-fs` is extremely important to remove false positives
- FFUF is one of the most powerful web fuzzing tools for enumeration and reconnaissance
