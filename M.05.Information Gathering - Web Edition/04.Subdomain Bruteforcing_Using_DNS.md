# 📌 Subdomain Bruteforcing (CPTS Quick Notes)

## 🔍 Definition
- Active recon technique
- Finds subdomains using wordlists

---

## ⚙️ Process
1. Wordlist selection (general / targeted / custom)
2. Append word → domain (dev.example.com)
3. DNS query (A / AAAA)
4. Validate results

---

## 🛠️ Important Tools
- dnsenum
- dnsrecon
- amass
- fierce
- assetfinder
- puredns

---

# 🚀 dnsenum (Most Important)

## 📌 Features
- DNS record enum (A, NS, MX, TXT)
- Zone transfer attempt (AXFR)
- Subdomain brute-force
- Google scraping
- Reverse DNS
- WHOIS lookup

---

## ⭐ MAIN COMMAND (EXAM IMPORTANT)

    dnsenum --enum inlanefreight.com \
    -f /usr/share/seclists/Discovery/DNS/subdomains-top1million-20000.txt \
    -r

---

## 📌 Command Breakdown
- `--enum` → enable full enumeration
- `-f` → wordlist for brute-force
- `-r` → recursive brute-force

---

## ⚡ Output Insight
- Finds subdomains:
    www.inlanefreight.com
    support.inlanefreight.com

---

# 🔥 AXFR (Zone Transfer)

## ⭐ MAIN COMMAND

    dig axfr @<nameserver> <domain>

## Example

    dig axfr @zonetransfer.me zonetransfer.me

---

## 📌 Key Points
- AXFR = full DNS dump
- Works only if misconfigured
- High-value vulnerability

---

# 🧠 Exam Tips
- Always try:
    dig NS <domain>

- Then:
    dig axfr @<NS> <domain>

- Use SecLists wordlist for brute-force
- Recursive (-r) = deeper enumeration

---

# ⚠️ Common Issues
- SERVFAIL → DNS misconfig / DNSSEC
- NXDOMAIN → domain not exist
- .htb → lab-only domains (VPN required)
