# 📌 Web Reconnaissance (Quick Notes)

## Definition
- Initial phase of pentesting (Information Gathering)
- Collect information about target (website/application)

## Purpose
- Identify assets (domains, IPs, subdomains)
- Discover hidden/sensitive information
- Analyze attack surface
- Gather intelligence for attacks/social engineering

---

# 📊 Penetration Testing Flow
1. Pre-Engagement  
2. Information Gathering  
3. Vulnerability Assessment  
4. Exploitation  
5. Post-Exploitation  
6. Lateral Movement  
7. Proof of Concept  
8. Post-Engagement  

---

# 🔍 Types of Reconnaissance

## 1. Active Recon
- Direct interaction with target
- High risk of detection

### Techniques
- **Port Scanning** → Identify open ports (Nmap)
- **Vulnerability Scanning** → Find vulnerabilities (Nessus, OpenVAS)
- **Network Mapping** → Discover network paths (traceroute)
- **Banner Grabbing** → Service info (Netcat, curl)
- **OS Fingerprinting** → Detect OS (Nmap -O)
- **Service Enumeration** → Service versions (Nmap -sV)
- **Web Spidering** → Crawl website (Burp Suite, OWASP ZAP)

---

## 2. Passive Recon
- No direct interaction with target
- Very low risk of detection

### Techniques
- **Search Engines** → Google dorks
- **WHOIS Lookup** → Domain information
- **DNS Analysis** → Subdomain enumeration (dig, nslookup)
- **Web Archives** → Wayback Machine
- **Social Media** → Employee/organization info
- **Code Repositories** → GitHub leaks

---

# ⚖️ Active vs Passive

| Feature         | Active Recon | Passive Recon |
|----------------|------------|--------------|
| Interaction    | Yes        | No           |
| Detection Risk | High       | Very Low     |
| Data Depth     | High       | Limited      |

---

# 🧠 Key Exam Points
- Recon = foundation of pentesting
- Passive = stealth, Active = detailed
- DNS + WHOIS = starting point
- Helps in attack surface mapping
