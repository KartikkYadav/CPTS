# 📌 Automating Recon (CPTS Notes)

## 🔍 Definition
- Use tools/scripts to automate **information gathering**
- Covers:
  - DNS
  - Subdomains
  - Crawling
  - Ports

---

## 🎯 Why Automation?

- ⚡ Efficiency → faster than manual
- 📈 Scalability → multiple targets
- 🎯 Consistency → no human error
- 🔍 Coverage → deeper recon
- 🔗 Integration → chain tools together

---

# 🛠️ Recon Frameworks (IMPORTANT)

## ⭐ FinalRecon
- Python tool
- Features:
  - Whois
  - SSL check
  - Headers
  - Crawling

---

## ⭐ Recon-ng
- Modular framework
- Features:
  - DNS enum
  - Subdomains
  - Port scan
  - Crawling

---

## ⭐ theHarvester (VERY IMPORTANT)

## MAIN COMMAND:

    theHarvester -d <domain> -b google

## Use:
- Emails
- Subdomains
- Employees

---

## ⭐ SpiderFoot
- OSINT automation tool
- Collects:
  - IPs
  - domains
  - emails
  - social profiles

---

## ⭐ OSINT Framework
- Collection of OSINT tools
- Manual + automated recon

---

# ⚡ Attack Flow (CPTS)

1. Start with:
    theHarvester

2. Expand with:
    Recon-ng / SpiderFoot

3. Combine with:
    dnsenum / amass / crawling

---

# 🔥 Key Commands

    theHarvester -d target.com -b google
    recon-ng
    finalrecon --help
