# 🧬 Fingerprinting (Brief Notes)

---

## 🔹 Concept

Fingerprinting = identifying **technologies used by a target**  
(e.g., server, CMS, frameworks)

👉 Goal: choose **targeted attacks**, not blind testing

---

## 🔍 Basic Enumeration

### Check Headers
    curl -I target

Find:
- Server (Apache/Nginx)
- Version
- Redirects

---

### Follow Redirects
    curl -I -L target

Find:
- Real domain (e.g., www)
- Backend technologies

---

## 🧠 Technology Identification

Use multiple clues together:

Example (WordPress):
- `X-Redirect-By: WordPress`
- `/wp-json`
- `/wp-login.php`

👉 Conclusion: WordPress detected

---

## 🛡️ WAF Detection

    wafw00f target

👉 If WAF present:
- Requests may be blocked
- Adjust attack strategy

---

## 🔎 Deeper Fingerprinting

    nikto -h target -Tuning b

Find:
- Outdated software
- Exposed files
- Missing security headers

---

## ⚠️ Important Findings

Focus on:

- Server version → check CVEs  
- CMS → defines attack surface  
- Sensitive endpoints → `/wp-login.php`  
- Misconfigurations → missing headers  
- WAF → affects testing  

---

## ⚡ Workflow

    curl -I target
    curl -I -L target
    curl target
    wafw00f target
    nikto -h target -Tuning b

---

## 🔥 Key Idea

Fingerprinting answers:

👉 **What is running?**  
👉 **What can I attack?**
