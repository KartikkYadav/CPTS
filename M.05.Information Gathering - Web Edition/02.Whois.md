# Utilizing WHOIS: Security Analysis & Implementation

WHOIS is a query and response protocol used for querying databases that store the registered users or assignees of an Internet resource, such as a domain name or an IP address block.

---

## 1. Case Studies: The Value of WHOIS Data

### Scenario 1: Phishing Investigation
An email security gateway flags a suspicious email claiming to be from the company's bank. A security analyst performs a WHOIS lookup on the linked domain and finds:
* **Registration Date:** The domain was registered just a few days ago.
* **Registrant:** Information is hidden behind a privacy service.
* **Name Servers:** Associated with a known "bulletproof" hosting provider.

**Outcome:** The combination of recent registration, hidden data, and suspicious hosting confirms a phishing campaign. The domain is blocked, and employees are warned.

### Scenario 2: Malware Analysis
A researcher analyzes malware communicating with a Command-and-Control (C2) server. WHOIS reveals:
* **Registrant:** An individual using a free, anonymous email service.
* **Location:** A country with high cybercrime prevalence.
* **Registrar:** Known for lax abuse policies.

**Outcome:** The researcher identifies the infrastructure as likely compromised or "bulletproof" and notifies the hosting provider to disrupt the C2 channel.

### Scenario 3: Threat Intelligence Report
Analysts track a sophisticated actor targeting financial institutions by clustering WHOIS data:
* **Patterns:** Domains are registered in groups shortly before attacks.
* **Identities:** Registrants use various aliases and fake identities.
* **Infrastructure:** Domains share the same name servers.

**Outcome:** Analysts create a detailed profile of the actor's Tactics, Techniques, and Procedures (TTPs) and generate Indicators of Compromise (IOCs) for defensive blocking.

## 2. Technical Implementation

### Installation
On a Linux system (such as Debian/Ubuntu), ensure the utility is installed:

    sudo apt update && sudo apt install whois -y

# WHOIS Enumeration Notes (CPTS)

## Usage Example
    whois facebook.com

---

## Key Output Data Points

- **Registrar**: RegistrarSafe, LLC  
- **Creation Date**: 1997-03-29  
- **Organization**: Meta Platforms, Inc.  
- **Name Servers**:  
  - A.NS.FACEBOOK.COM  
  - B.NS.FACEBOOK.COM  
- **Domain Status**:  
  - clientTransferProhibited  
  - serverDeleteProhibited  
  → Indicates strong security protections  

---

## Analysis Summary (CPTS)

### Domain Longevity
- Old domains → More trustworthy  
- Newly registered → High risk indicator  

### Ownership
- Identifies parent organization  
- Useful for attribution (e.g., Meta Platforms, Inc.)  

### Domain Status (EPP Codes)
- Shows protection level  
- Prevents:
  - Unauthorized transfer  
  - Deletion  
  - Modification  

### DNS Management
- Internal NS (e.g., ns.facebook.com)  
→ Organization controls its own DNS infrastructure  

### Red Flags
- WHOIS privacy protection (hidden data)  
- Free email providers (e.g., Gmail, Yahoo)  
- Bulletproof hosting providers  

---

## Conclusion

- WHOIS gives:
  - Ownership info  
  - Domain metadata  
  - Infrastructure hints  

- Limitations:
  - GDPR / privacy redaction  
  - Incomplete data  

### Combine with:
- DNS Enumeration  
- Passive DNS  
- IP Footprinting
