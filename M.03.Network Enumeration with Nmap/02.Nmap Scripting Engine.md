# 📌 Nmap Scripting Engine (NSE) – CPTS Notes

---

## 🔹 What is NSE?

- Nmap Scripting Engine (NSE) allows execution of **Lua scripts**
- Used for:
  - Enumeration
  - Service interaction
  - Vulnerability detection
- Enhances Nmap beyond basic port scanning  

---

## 🧠 NSE Script Categories (Important)

| Category   | Purpose |
|-----------|--------|
| auth | Finds authentication credentials |
| broadcast | Host discovery via broadcast |
| brute | Brute-force attacks |
| default | Default scripts (`-sC`) |
| discovery | Service enumeration |
| dos | DoS testing (⚠️ risky) |
| exploit | Exploits known vulnerabilities |
| external | Uses external services |
| fuzzer | Sends malformed inputs |
| intrusive | May affect system stability |
| malware | Detects malware |
| safe | Non-intrusive scripts |
| version | Service version detection |
| vuln | Vulnerability detection |

---

## ⚙️ NSE Usage Commands

### 🔹 Default Scripts
    nmap <target> -sC

✔ Runs common safe scripts

---

### 🔹 Run Scripts by Category
    nmap <target> --script <category>

✔ Example:
    nmap <target> --script vuln

---

### 🔹 Run Specific Scripts
    nmap <target> --script script1,script2

✔ Example:
    nmap 10.129.2.28 -p 25 --script banner,smtp-commands

---

## 🔍 Example: SMTP Enumeration

### Command:
    nmap 10.129.2.28 -p 25 --script banner,smtp-commands

### Key Findings:
- Banner reveals OS → Ubuntu
- SMTP Commands:
  - VRFY → user enumeration possible
  - STARTTLS supported


## ⚡ Aggressive Scan (-A)

### Command:
    nmap <target> -A

### Includes:
- -sV → Version detection  
- -O → OS detection  
- --traceroute  
- -sC → Default scripts  

---


## 🛡️ Vulnerability Scanning

### Command:
    nmap <target> -p 80 -sV --script vuln

---

### 🔍 What it does:
- Detects:
  - Web directories
  - Application versions
  - Users
- Maps:
  - CVEs
  - Known vulnerabilities

---

### 🧠 Example Findings:
- `/wp-login.php` → Login page
- Username → admin
- CVEs:
  - CVE-2019-0211
  - CVE-2018-1312
