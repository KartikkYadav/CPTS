# 🔐 Firewall & IDS/IPS Evasion Labs (Easy → Hard)

## 📌 Overview
These labs simulate real-world scenarios where:
- Systems are protected by **Firewall + IDS/IPS**
- Security improves after each test
- Goal is to **bypass detection and extract information**

---

# 🧪 Lab 1: Easy Lab

## 🎯 Objective
Identify the **Operating System** of the target machine.

## 🧠 Approach
- Perform **service/version enumeration**
- OS details often leak via service banners

## ⚙️ Command

    nmap -sV <IP>

## ✅ Result
- Output reveals OS → **Ubuntu**

## 💡 Key Concept
- Service version detection (`-sV`) can expose OS info via banners.

---

# 🧪 Lab 2: Medium Lab

## 🎯 Objective
Identify the **DNS server version** running on the target.

## ⚠️ Challenge
- Firewall blocks normal TCP scans
- Must use **UDP scan** (given in question)

## 🧠 Approach

### Step 1: Try TCP scan (fails)

    nmap -p 53 -sV <IP>

❌ No useful result due to firewall filtering

---

### Step 2: Use UDP scan (bypass firewall)

    nmap -p 53 -sU <IP>

## ✅ Result
- DNS service responds
- Version information obtained

## 💡 Key Concepts

| Concept | Explanation |
|--------|------------|
| UDP Scan (`-sU`) | Bypasses TCP-based firewall rules |
| Port 53 | DNS runs on UDP |
| IDS/IPS Behavior | May allow UDP while blocking TCP |

# 🧪 Lab 3: Hard Lab

## 🎯 Objective
Identify the **version of a newly added service** and retrieve the flag.

## 🧠 Hint Analysis
> “Service critical for handling large amounts of data”

👉 Likely:
- Custom service
- Running on a **non-standard port**

---

## ⚙️ Step 1: Full Port Scan

    nmap -p- -Pn -sV <IP>

### 🔍 Why?
| Flag | Purpose |
|------|--------|
| `-p-` | Scan all 65535 ports |
| `-Pn` | Skip host discovery (firewall bypass) |
| `-sV` | Detect service version |

---

## ✅ Result
- New service found on:

    Port: 50000

---

## ⚙️ Step 2: Connect to Service

    sudo ncat -nv --source-port 53 <IP> 50000

---

## 🔥 Why `--source-port 53`?

| Reason | Explanation |
|--------|------------|
| Firewall Bypass | Port 53 (DNS) is trusted |
| Evasion Technique | Traffic appears legitimate |
| IDS/IPS Evasion | Reduces detection likelihood |

---

## ⚠️ Note
- May fail on local machine (Parrot/Kali)
- Works on **Pwnbox / VPN environment**
- Requires `sudo` due to privileged port usage

---

## ✅ Result
- Connection successful
- **Flag retrieved from service**

---

# 🧠 Key Takeaways (CPTS Important 🔥)

## 🔹 Enumeration Strategy
- Always scan **all ports** when services are hidden:

    nmap -p- -Pn -sV <IP>

---

## 🔹 Firewall Evasion Techniques

| Technique | Command |
|----------|--------|
| Skip host discovery | `-Pn` |
| UDP scan | `-sU` |
| Trusted port spoofing | `--source-port 53` |
| Full port scan | `-p-` |

---

## 🔹 Real-World Insight
- Firewalls often:
  - Block ICMP (ping)
  - Filter TCP
  - Allow DNS (UDP 53)

👉 Attackers exploit **trusted ports** to bypass restrictions.

---

## 🧠 Memory Cheatsheet

- `-Pn` → Assume host is alive  
- `-p-` → Scan everything  
- `-sU` → Use UDP (bypass firewall)  
- `--source-port 53` → Look like DNS traffic
