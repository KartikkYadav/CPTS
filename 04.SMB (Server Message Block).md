# SMB (Server Message Block) – CPTS Quick Notes

## 🔹 What is SMB
- Protocol for sharing:
  - Files
  - Folders
  - Printers
- Works in **client-server model**
- Mainly used in **Windows**
- Linux uses **Samba** for SMB

---

## 🔹 How SMB Works
- Client → requests resource
- Server → provides access
- Uses **TCP (3-way handshake)**
- Communication happens over:
  - **445 (modern SMB)**
  - **137–139 (old NetBIOS)**

---

## 🔹 SMB Versions
- SMB1 (CIFS) → Old, insecure ❌
- SMB2/3 → Faster, secure ✅

---

## 🔹 Samba (Linux SMB)
- Implements SMB on Linux
- Daemons:
  - `smbd` → file sharing
  - `nmbd` → NetBIOS

---

## 🔹 Important Concepts

### Share
- A folder exposed over network  
- Example:
    //IP/notes

---

### Permissions (ACL)
- Control access:
  - Read
  - Write
  - Execute

---

### Workgroup
- Group of machines in network

---

### NetBIOS
- Name-based communication (instead of IP)

---

## 🔹 Important Config File

    /etc/samba/smb.conf

### Common Settings

| Setting      | Meaning |
|-------------|--------|
| workgroup   | Network name |
| path        | Shared folder |
| browseable  | Visible to users |
| read only   | Disable writing |
| guest ok    | Allow no-password login |

---

## 🔥 Dangerous Settings (Exam Important)

| Setting              | Risk |
|---------------------|------|
| browseable = yes    | Attacker can list shares |
| guest ok = yes      | No authentication |
| writable = yes      | Upload malicious files |
| create mask = 0777  | Full permissions |

👉 Misconfiguration = easy attacker access

---

## 🔹 Enumeration Commands

### 1. Nmap Scan
    nmap -sC -sV -p139,445 <IP>

---

### 2. List Shares (Anonymous)
    smbclient -N -L //<IP>

---

### 3. Connect to Share
    smbclient //<IP>/notes

Commands inside:
    ls
    get file.txt

---

### 4. Run Local Commands
    !ls
    !cat file.txt

## 🔥 Key Attack Points (CPTS)

- Anonymous login → HIGH VALUE  
- Writable share → upload shell  
- User enumeration → brute force creds  
- Sensitive files → creds/config leaks  

---

## 🧠 Exam Mindset

1. Check SMB ports (139, 445)  
2. Try anonymous access  
3. List shares  
4. Access shares  
5. Download files  
6. Enumerate users  
7. Look for credentials
