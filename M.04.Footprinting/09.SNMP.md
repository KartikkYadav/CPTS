# SNMP (Simple Network Management Protocol)

## Overview

SNMP is used to:
- Monitor network devices
- Manage configurations remotely
- Collect system information

Common SNMP-enabled devices:
- Routers
- Switches
- Servers
- Printers
- IoT devices

:contentReference[oaicite:0]{index=0}

---

# SNMP Ports

| Port | Purpose |
|------|----------|
| UDP 161 | SNMP communication |
| UDP 162 | SNMP traps |

---

# SNMP Traps

SNMP traps are:
```text
Automatic notifications sent from server to client
```

Used when:
- Errors occur
- Events are triggered
- Alerts generated

---

# MIB (Management Information Base)

MIB is:
```text
Database of SNMP objects
```

Contains:
- OIDs
- Object names
- Data types
- Access permissions

---

# OID (Object Identifier)

OID uniquely identifies SNMP objects.

Example:

```text
1.3.6.1.2.1.1.5.0
```

OIDs are arranged in:
```text
Hierarchical tree structure
```

---

# SNMP Versions

| Version | Security |
|---------|-----------|
| SNMPv1 | No encryption/authentication |
| SNMPv2c | Community-string based |
| SNMPv3 | Authentication + Encryption |

---

# SNMPv1

Features:
- Basic monitoring
- No encryption
- No authentication

Problem:
```text
Data sent in plain text
```

---

# SNMPv2c

Uses:
```text
Community Strings
```

Example:

```text
public
private
```

Still vulnerable because:
- Community strings transmitted in cleartext

---

# SNMPv3

Adds:
- Username/password authentication
- Encryption
- Better security

More secure but:
- More complex to configure

---

# Community Strings

Community strings act like:
```text
Passwords
```

Common defaults:

```text
public
private
```

---

# Default SNMP Configuration

## View Configuration

```bash
cat /etc/snmp/snmpd.conf | grep -v "#" | sed -r '/^\s*$/d'
```

---

# Important Dangerous Settings

| Setting | Risk |
|---------|------|
| `rwuser noauth` | Full access without auth |
| `rwcommunity public` | Read/write access |
| `rwcommunity6` | IPv6 read/write access |

# Footprinting SNMP

Useful tools:
- `snmpwalk`
- `onesixtyone`
- `braa`

# SNMPWalk

## Main Command

```bash
snmpwalk -v2c -c public 10.129.14.128
```

---

# Command Breakdown

| Option | Purpose |
|--------|----------|
| `-v2c` | SNMP version |
| `-c public` | Community string |


# Brute-Force Community Strings

```bash
onesixtyone -c /opt/useful/seclists/Discovery/SNMP/snmp.txt 10.129.14.128
```

```text
Community string = public
```

# Important Risks

Misconfigured SNMP may expose:
- System details
- Installed packages
- Network information
- Usernames
- Email addresses
- Service versions

---

# Key Points

- SNMP is widely used for monitoring devices
- SNMPv1/v2c are insecure
- Community strings often weak/default
- SNMP enumeration reveals valuable internal information
- `snmpwalk`, `onesixtyone`, and `braa` are common tools

---

# One-Line Revision

SNMP is a network management protocol that can leak sensitive system information through weak community strings and insecure configurations, especially in SNMPv1 and SNMPv2c.
