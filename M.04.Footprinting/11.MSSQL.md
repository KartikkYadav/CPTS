# MSSQL (Microsoft SQL Server)

## Overview

MSSQL is:
- Microsoft’s relational database management system
- SQL-based database server
- Commonly used in Windows environments

Popular with:
- Enterprise applications
- .NET applications
- Active Directory environments

:contentReference[oaicite:0]{index=0}

---

# Default MSSQL Port

| Port | Service |
|------|----------|
| TCP 1433 | MSSQL |

---

# MSSQL Characteristics

- Closed source
- Developed by Microsoft
- Commonly runs on Windows
- Supports Windows Authentication
- Integrates with Active Directory

| Database | Purpose |
|----------|----------|

---

# Default Authentication

MSSQL commonly uses:
```text
Windows Authentication
```

This means:
- Credentials validated through Windows
- Uses local SAM or Active Directory

---

# Important Security Risk

If a Windows account is compromised:
- Database access may also be compromised
- Privilege escalation possible
- Lateral movement possible

---

# Dangerous MSSQL Settings

| Setting | Risk |
|---------|------|
| No encryption | Credentials sniffing |
| Self-signed certificates | Spoofing risk |
| Named pipes enabled | Additional attack surface |
| Weak `sa` password | Full DB compromise |

---

# Named Pipes

Example:

```text
\\10.129.201.248\pipe\sql\query
```

Named pipes allow:
- Local/remote communication
- Alternate MSSQL access methods

---

# Nmap MSSQL Enumeration

## Main Command

```bash
sudo nmap --script ms-sql-info,ms-sql-empty-password,ms-sql-xp-cmdshell,ms-sql-config,ms-sql-ntlm-info,ms-sql-tables,ms-sql-hasdbaccess,ms-sql-dac,ms-sql-dump-hashes --script-args mssql.instance-port=1433,mssql.username=sa,mssql.password=,mssql.instance-name=MSSQLSERVER -sV -p 1433 10.129.201.248
```

---

# Information Gathered

Nmap can reveal:
- MSSQL version
- Hostname
- Instance name
- Named pipes
- Empty passwords
- NTLM information

---

# Example Findings

```text
Microsoft SQL Server 2019
Instance: MSSQLSERVER
Named pipe: \\10.129.201.248\pipe\sql\query
```

# Why MSSQL Matters in Pentesting

MSSQL compromise may lead to:
- Domain compromise
- Sensitive data theft
- Command execution
- Lateral movement

Especially dangerous in:
```text
Active Directory environments
```

---

# Key Points

- MSSQL commonly runs on Windows
- Uses TCP port 1433
- Supports Windows Authentication
- Nmap and Metasploit support MSSQL enumeration
- `mssqlclient.py` is useful for pentesting
- Weak `sa` credentials are highly dangerous

# One-Line Revision

MSSQL is Microsoft’s SQL database server commonly used in Windows environments and can expose critical information or lead to domain compromise through weak authentication and insecure configurations.
