# 📌 DNS – CPTS Quick Notes

## 🔹 What is DNS
- Resolves **domain → IP**
- Distributed system (no central DB)
- Works like a **phonebook**

---

## 🔹 DNS Server Types
- **Root Server** → Handles TLDs (.com, .net)
- **Authoritative NS** → Final answer for domain
- **Non-Authoritative** → Cached answers
- **Caching Server** → Stores responses
- **Resolver** → Client-side lookup

---

## 🔹 Important DNS Records

| Record | Purpose |
|--------|--------|
| A | Domain → IPv4 |
| AAAA | Domain → IPv6 |
| MX | Mail server |
| NS | Nameserver |
| TXT | SPF, verification |
| CNAME | Alias |
| PTR | Reverse lookup |
| SOA | Zone info |

---

# 🔹 🔥 DNS Enumeration (MOST IMPORTANT)

## 1️⃣ NS Enumeration
    dig ns target.com @DNS_IP

👉 Find **nameservers**

---

## 2️⃣ Version Disclosure
    dig CH TXT version.bind DNS_IP

👉 Reveals **BIND version** (if enabled)

---

## 3️⃣ ANY Query (Info Dump)
    dig any target.com @DNS_IP

👉 Dumps available records (TXT, NS, SOA)

---

## 4️⃣ Zone Transfer (CRITICAL 🔥)
    dig axfr target.com @DNS_IP

👉 If misconfigured → **FULL DNS dump**

Internal zone:
    dig axfr internal.target.com @DNS_IP

⚠️ Misconfig → leaks:
- Internal IPs
- Hostnames
- Services

---

## 5️⃣ Subdomain Brute Force (Manual)
    for sub in $(cat wordlist.txt); do
      dig $sub.target.com @DNS_IP
    done

---

## 6️⃣ Subdomain Brute Force (Tool)
    dnsenum --dnsserver DNS_IP --enum -f wordlist.txt target.com

---

# 🔹 🔥 Key Misconfigurations

| Setting | Risk |
|--------|------|
| allow-transfer | Zone dump possible |
| allow-recursion | Abuse for attacks |
| allow-query | Open resolver |
| zone-statistics | Info leakage |

---

# 🔹 🔥 Zone Transfer Concept
- Uses **TCP 53**
- Sync between **master ↔ slave**
- If open → attacker gets **entire zone**

---

## 🔹 🔥 Reverse Lookup
- IP → Domain using **PTR**
- Stored in reverse zone files

---

## 🔹 🔥 Key Files (BIND)
- `/etc/bind/named.conf`
- `/etc/bind/named.conf.local`
- Zone files: `/etc/bind/db.domain.com`

---

# 🧠 Exam Tips (CPTS)
- Always try:
  - `dig ns`
  - `dig any`
  - `dig axfr` ⭐
- Check for:
  - Zone transfer
  - Subdomains
  - Internal records
- Use **dnsenum / wordlists**
