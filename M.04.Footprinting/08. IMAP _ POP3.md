# IMAP / POP3

## Overview

IMAP and POP3 are email retrieval protocols used to access emails from a mail server.

| Protocol | Purpose |
|----------|----------|
| IMAP | Manage emails on server |
| POP3 | Retrieve/download emails |

# IMAP

## Overview

IMAP:
- Stores emails on server
- Supports folder structures
- Allows synchronization across devices
- Supports multiple mailboxes

Emails remain on server until deleted.

---

# IMAP Ports

| Port | Service |
|------|----------|
| 143 | IMAP |
| 993 | IMAPS (SSL/TLS) |

---

# Important Features

- Folder management
- Online synchronization
- Multi-client access
- Server-side mailbox handling

---

# POP3

## Overview

POP3 is simpler than IMAP.

Functions:
- Retrieve emails
- List emails
- Delete emails

Usually downloads emails locally.

---

# POP3 Ports

| Port | Service |
|------|----------|
| 110 | POP3 |
| 995 | POP3S (SSL/TLS) |

---

# Important Difference

| IMAP | POP3 |
|------|------|
| Emails stay on server | Emails usually downloaded |
| Folder support | Limited features |
| Multi-device sync | Basic retrieval only |

---

# Common IMAP Commands

| Command | Purpose |
|---------|----------|
| `LOGIN user pass` | Login |
| `LIST "" *` | List mailboxes |
| `SELECT INBOX` | Open mailbox |
| `FETCH <ID> all` | Read email |
| `LOGOUT` | Close session |

---

# Common POP3 Commands

| Command | Purpose |
|---------|----------|
| `USER` | Username |
| `PASS` | Password |
| `LIST` | List emails |
| `RETR id` | Retrieve email |
| `DELE id` | Delete email |
| `QUIT` | Exit session |

---

# Dangerous Configurations

Misconfigurations may expose:
- Credentials
- Email contents
- Authentication logs

---

# Important Dangerous Settings

| Setting | Purpose |
|---------|----------|
| `auth_debug` | Authentication debug logging |
| `auth_verbose` | Verbose auth failures |
| `auth_verbose_passwords` | Logs passwords |
| `auth_anonymous_username` | Anonymous login support |

# Footprinting Mail Services

## Nmap Scan

```bash
sudo nmap 10.129.14.128 -sV -p110,143,993,995 -sC
```

---

# Information Gathered

- Mail server software
- SSL certificates
- Capabilities
- Organization name
- Common Name (CN)

# IMAP Access with cURL

## Main Command

```bash
curl -k 'imaps://10.129.14.128' --user user:p4ssw0rd
```

---

# Purpose

- Connect to IMAPS
- Authenticate user
- List mailboxes

---

# OpenSSL Interaction

## POP3S Connection

```bash
openssl s_client -connect 10.129.14.128:pop3s
```

---

## IMAPS Connection

```bash
openssl s_client -connect 10.129.14.128:imaps
```

---

# Useful Information From SSL

- Certificate details
- TLS version
- Cipher suites
- Mail server banner

# Example Credentials

```text
robin:robin
```

---

# Why This Matters

If valid credentials are discovered:
- Emails can be read
- Sensitive information may leak
- Internal communication exposed
- Password reset emails accessible

---

# Key Points

- IMAP manages emails on server
- POP3 mainly downloads emails
- SSL/TLS commonly used on ports 993 and 995
- Nmap helps enumerate mail services
- cURL and OpenSSL can interact with mail servers
- Weak credentials can expose sensitive mailboxes

---

# One-Line Revision

IMAP and POP3 are email access protocols that can be enumerated and interacted with using tools like Nmap, cURL, and OpenSSL to identify mail services, capabilities, and potentially access sensitive emails.
