# IPMI (Intelligent Platform Management Interface)

## Overview

IPMI is a hardware-based management protocol used for:
- Remote server management
- Monitoring systems
- Power control
- BIOS management
- Recovery after crashes

It works independently from:
- Operating system
- BIOS
- CPU state

This means administrators can manage systems even when:
- The OS is down
- The server is powered off
- The system is unresponsive

# BMC (Baseboard Management Controller)

The BMC is:
```text
The core management controller used by IPMI
```

It provides:
- Remote management
- Hardware access
- Out-of-band administration

Always try:

- Vendor defaults

because admins often forget to change them.

---

# Dangerous IPMI Issue

IPMI 2.0 contains a flaw in:
```text
RAKP Authentication Protocol
```

The server sends:
- Salted SHA1/MD5 password hashes

before authentication succeeds.

---

# Why This Is Dangerous

Attackers can:
1. Request hashes remotely
2. Crack hashes offline
3. Reuse credentials elsewhere

without valid authentication.

# Why IPMI is Critical

IPMI exposure can lead to:
- Full server compromise
- Credential reuse attacks
- Lateral movement
- Persistence
- Remote administration abuse

# Key Points

- IPMI provides hardware-level server management
- Uses UDP port 623
- BMC compromise is extremely dangerous
- Default credentials are common
- IPMI 2.0 leaks password hashes through RAKP
- Metasploit supports IPMI enumeration and hash dumping
