# MySQL

## Overview

MySQL is an:
- Open-source relational database management system (RDBMS)
- Developed and supported by Oracle

Used to:
- Store structured data
- Manage web application data
- Process SQL queries

---

# Important Configuration Entries

| Setting | Purpose |
|---------|----------|
| `user` | MySQL service user |
| `password` | MySQL credentials |
| `port` | Listening port |
| `datadir` | Database storage location |
| `secure_file_priv` | Restricts file import/export |

---

# Dangerous Settings

| Setting | Risk |
|---------|------|
| `debug` | Verbose error output |
| `sql_warnings` | Information leakage |
| `secure_file_priv` | File read/write abuse |
| Plaintext passwords | Credential exposure |

---

# Why MySQL is Dangerous

Misconfigured MySQL may expose:
- Credentials
- Databases
- User information
- Password hashes
- Sensitive application data

---

# Nmap MySQL Enumeration

## Main Command

```bash
sudo nmap 10.129.14.128 -sV -sC -p3306 --script mysql*
```

---

# What Nmap Can Detect

- MySQL version
- Empty passwords
- Valid users
- Authentication methods
- Vulnerabilities

---

# Example Findings

```text
root:<empty> - Valid credentials
Version: 8.0.26
```

---

# Important Note

Nmap results may contain:
```text
False Positives
```

Always manually verify findings.

# Connect with Password

## Main Command

```bash

```

---

# Important Syntax

No space between:
```text
-p
```

and password.

Correct:

```bash
-pPassword
```

Wrong:

```bash
-p Password
```

# Key Points

- MySQL stores structured application data
- Runs on TCP port 3306
- Commonly used in LAMP/LEMP stacks
- Misconfigurations can expose sensitive data
- Nmap supports MySQL enumeration scripts
- `mysql` client used for direct interaction

MySQL is a relational database system commonly used by web applications that can expose sensitive data through weak credentials, verbose errors, and insecure configurations.
